## 週次の主要機能リリース

### W50 (12/08-14) — プロジェクト立ち上げ・基盤構築
- ターミナル統合フロントエンドを実装（XTerm.js）
- PTYマネージャーとターミナルWebSocket統合
- リッチ出力表示機能（マークダウン・シンタックスハイライト）
- ライト/ダークモード切り替え機能

### W51 (12/15-21) — マルチセッション・UI強化
- セッション作成時のモデル選択と一括作成機能
- モバイルUI最適化
- ESLint v9へのアップグレード
- /projects から / へのリダイレクト

### W52 (12/22-28) — CLI・プロセス管理
- CLI にデーモンモード（start/stop/status/logs）を追加
- npx claude-work で自動セットアップ
- ブラウザ通知システムの実装
- プロセス状態確認API・再起動API実装
- GitStatusBadgeコンポーネント統合

### W01 (12/29-01/04) — アーキテクチャ刷新
- 薄いラッパーUIアーキテクチャへの移行
- UX改善と認証機能削除
- セッション管理機能の改善

### W02 (01/05-11) — Docker導入
- Docker統合機能を追加

### *(W03 休止)*

### W04 (01/19-25) — 実行環境基盤
- 実行環境（Execution Environments）機能の実装
- Terminal WebSocketのDocker環境サポート

### W05 (01/26-02/01) — リモートリポジトリ対応
- リモートGitリポジトリの直接登録機能
- 左メニューUI改善
- npx github:... による直接インストール対応

### W06 (02/02-08) — Drizzle移行・永続化
- Prisma → Drizzle ORM移行
- セッション永続化と複数ブラウザ共有の改善
- DATA_DIR環境変数でデータディレクトリ永続化
- Claudeターミナルの操作性改善（フォーカス/コピペ/改行/画像添付）
- HOST環境変数で外部アクセス許可

### W07 (02/09-15) — 認証・マイグレーション
- GitHub PAT認証によるHTTPS clone対応
- Docker環境プロジェクトのworktree作成機能
- PRAGMA user_versionを使った自動DBマイグレーション
- Claude Code実行時のCLIオプション・環境変数設定機能

### W08 (02/16-22) — Dockerode移行・本番化
- Docker CLI → Dockerode Node.jsライブラリ移行（Phase 1-4）
- アプリケーションDockerイメージ公開とGitHub Release自動化
- Docker環境にポートマッピング・ボリュームマウント設定
- Docker主体+リモートリポジトリ対応への移行
- Developer Tools Settings管理機能

### W09 (02/23-03/01) — UX改善・Docker本番化
- Docker Composeをプレビルドイメージに移行
- Claude Code設定の永続化をDocker名前付きVolume
- プロジェクト追加ウィザード形式への変更
- Docker Volumeの自動命名と既存Volume選択
- プロジェクト-環境紐付けの一元化
- ENCRYPTION_KEY自動生成・永続化
- DBマイグレーション v6

### W10 (03/02-05) — セキュリティ・拡張性
- Docker環境ネットワークフィルタリング機能
- デフォルト環境の概念を廃止し環境選択を必須化
- 環境/プロジェクト削除時にDocker Volume保持オプション
- 言語別Docker拡張イメージとCI/CDマトリクスビルド
