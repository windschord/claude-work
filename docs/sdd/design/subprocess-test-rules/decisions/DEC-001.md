# DEC-001: cli.tsテストのモック戦略

## 決定

cli.tsのテストでは、`child_process`をモックし、各関数を動的importで呼び出す。

## 背景

`cli.ts`はモジュールトップレベルで以下の副作用がある:
- `__dirname`から`projectRoot`を計算
- `process.argv`を参照してコマンドを決定
- `resolvePm2Cmd()`でファイルシステムを走査

これらの副作用があるため、単純なimportではテストが困難。

## 選択肢

### A) cli.tsを直接テスト（動的import） - 採用

各関数をexportしてテスト可能にし、モジュールの動的importで副作用を制御する。

利点:
- 実際のコードパスをテストできる
- cwdの値を直接検証可能

欠点:
- モジュール副作用の制御が複雑

### B) 関数をcli-utils.tsに抽出してテスト - 不採用

spawnSync呼び出しを含む関数をcli-utils.tsに移動する。

利点:
- テストが簡単
- 副作用を分離できる

欠点:
- 既存コードの大規模リファクタリングが必要
- Issue #111のスコープを超える

## 結論

cli.tsの関数をexportし、テスト時はモックと動的importで副作用を制御する。大規模リファクタリングは避け、最小限の変更でcwd/envのアサーションを追加する。
