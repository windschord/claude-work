# 非機能要件: ユーザビリティ

## NFR-101: 直感的なUI

システムは、ポートマッピングとボリュームマウントの設定を動的に追加・削除できるリスト形式のUIを提供しなければならない。各行に入力フィールドと削除ボタンを配置し、「追加」ボタンで新しい行を追加できるようにする。

## NFR-102: デフォルト値の提供

- ポートマッピング: プロトコルのデフォルトはTCPとする
- ボリュームマウント: アクセスモードのデフォルトはrw（読み書き）とする

## NFR-103: リアルタイムバリデーション

システムは、入力値の変更時にリアルタイムでバリデーションを実行し、即座にエラーメッセージを表示しなければならない。フォーム送信時ではなく、入力時に検証を行う。

## NFR-104: 設定のプレビュー

システムは、設定変更時に影響を受ける実行中セッションの数を環境編集画面に表示しなければならない。

## NFR-105: 空状態の表示

ポートマッピング・ボリュームマウントが未設定の場合、システムは「設定なし」のメッセージと「追加」ボタンを表示しなければならない。
