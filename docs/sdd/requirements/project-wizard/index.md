# 要件定義: プロジェクト追加ウィザード

## 概要

現在のプロジェクト追加フロー（単一モーダルのAddProjectModal）を、ステップバイステップのウィザード形式に変更する。設定項目を4つのステップ（環境選択→認証情報設定→リポジトリ設定→セッション開始）に分割し、初回利用時でも迷わずプロジェクトを追加できるUIを提供する。

**関連Issue**: #173

## ユーザーストーリー一覧

| ID | タイトル | 優先度 | ステータス |
|----|---------|--------|-----------|
| US-001 | ウィザード形式でのプロジェクト追加 | 高 | 承認済 |
| US-002 | 環境選択ステップ（Step 1） | 高 | 承認済 |
| US-003 | 認証情報設定ステップ（Step 2） | 高 | 承認済 |
| US-004 | リポジトリ設定ステップ（Step 3） | 高 | 承認済 |
| US-005 | セッション開始ステップ（Step 4） | 中 | 承認済 |

## 機能要件サマリ

### US-001: ウィザード全体 (REQ-001-001〜008)
- ウィザードモーダル表示、4ステッププログレスバー、バリデーション後遷移、戻るボタンで値保持、プログレスバークリック遷移、次へボタン無効化、ESC/オーバーレイで閉じる、再オープン時リセット

### US-002: 環境選択 (REQ-002-001〜007)
- 環境一覧Listbox表示、ローディング表示、選択環境の概要表示、新しい環境を作成リンク、環境未存在時案内、環境未選択時次へ無効化、デフォルト環境初期選択

### US-003: 認証情報設定 (REQ-003-001〜007)
- PAT選択とSSH Key情報表示、アクティブPATドロップダウン、PAT作成ダイアログ、SSH認証オプション、スキップ可能、任意ステップ案内、PAT作成後自動選択

### US-004: リポジトリ設定 (REQ-004-001〜009)
- リポジトリタイプ選択、ローカルパス入力、リモートURL入力+保存場所選択、Docker/ホストオプション、HOST無効時Docker限定、プロジェクト名自動検出、空入力時次へ無効化、HTTPS+Docker時PAT自動適用

### US-005: セッション開始 (REQ-005-001〜007)
- 成功後ステップ表示、セッション名デフォルト値、セッション作成して開始、スキップしてプロジェクト一覧へ、失敗時エラー表示、作成中ローディング、選択環境のセッション適用

## 依存関係

- 既存API: POST /api/projects, POST /api/projects/clone, GET /api/github-pat, GET /api/environments
- 既存フック: useAppStore, useGitHubPATs, useEnvironments
- 既存コンポーネント: PATCreateDialog

## スコープ外

- API層の変更
- 環境作成のインラインフォーム
- SSH接続テスト機能
