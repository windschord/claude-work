# US-003: プロジェクトの環境変更制限

## ユーザーストーリー

**私は** ClaudeWorkのユーザーとして
**〜したい** プロジェクトの実行環境をセッションがない場合にのみ変更したい
**なぜなら** 既存のセッションが異なる環境のworktreeやコンテナを使用しているため、環境を変更すると不整合が生じるため

## 受入基準（EARS記法）

### AC-001: プロジェクト設定画面での環境表示

- プロジェクト設定画面において、システムは紐付いた環境名（コンテナ名）を表示しなければならない

### AC-002: セッション0件時の環境変更

- プロジェクト内のセッションが0件の場合、環境変更ボタンを有効化しなければならない
- 環境変更ボタンをクリックした時、利用可能な環境の選択ドロップダウンを表示しなければならない
- 環境を変更しプロジェクト更新APIを呼び出した時、システムはenvironment_idを更新しなければならない

### AC-003: セッション存在時の環境変更拒否

- もしプロジェクト内にセッションが1件以上存在するならば、環境変更ボタンを無効化し「セッションが存在するため変更できません」と表示しなければならない
- プロジェクト更新APIでenvironment_idの変更が要求された時、もしセッションが存在するならば、システムは409 Conflictを返さなければならない

## 優先度

High

## 画面/UI要件

- プロジェクト設定画面に「実行環境」セクションを表示
  - 環境名の表示（読み取り専用）
  - 「変更」ボタン（条件付き有効化）
  - セッション存在時は無効化理由メッセージを表示
