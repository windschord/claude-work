# US-001: プロジェクト作成時の環境必須指定

## ユーザーストーリー

**私は** ClaudeWorkのユーザーとして
**〜したい** プロジェクト作成時に使用する実行環境（コンテナ）を明示的に選択したい
**なぜなら** どの環境でセッションが実行されるかを事前に把握し、意図しない環境での実行を防ぎたいため

## 受入基準（EARS記法）

### AC-001: 環境選択UIの表示

- プロジェクト作成画面において、システムは利用可能な環境の選択ドロップダウンを表示しなければならない
- ドロップダウンには環境名と環境タイプ（DOCKER/HOST/SSH）を表示しなければならない
- 初期状態では環境が未選択であり、プロジェクト作成ボタンは無効化されていなければならない

### AC-002: 環境指定の必須化

- プロジェクト作成APIにenvironment_idが未指定の場合、システムは400エラーを返さなければならない
- エラーレスポンスには「実行環境の指定は必須です」というメッセージを含めなければならない

### AC-003: 環境が存在しない場合

- 環境が1つも登録されていない場合、プロジェクト作成画面はプロジェクト作成ボタンを無効化し、「利用可能な実行環境がありません。先に環境を作成してください。」と表示しなければならない

### AC-004: DBスキーマ変更

- projectsテーブルのenvironment_idカラムはNOT NULL制約を持たなければならない
- onDelete動作はRESTRICT（またはアプリレベルで同等のチェック）としなければならない

## 優先度

High

## 画面/UI要件

- プロジェクト作成モーダルに「実行環境」セレクトボックスを追加
- 環境はタイプ別（DOCKER -> HOST -> SSH）にソートして表示
- デフォルト環境がある場合は初期選択（ただしユーザーが変更可能）

## データ要件

- `projects.environment_id`: nullable -> NOT NULL に変更
- `projects.environment_id`の外部キー制約: `onDelete: 'set null'` -> `onDelete: 'restrict'`（またはアプリレベルチェック）
