# US-002: 使用中環境の削除防止

## ユーザーストーリー

**私は** ClaudeWorkの管理者として
**〜したい** プロジェクトで使用中の環境を誤って削除できないようにしたい
**なぜなら** 環境を削除すると紐付いたプロジェクトのセッション実行が不可能になり、復旧が困難なため

## 受入基準（EARS記法）

### AC-001: 削除API のチェック

- 環境削除APIが呼び出された時、システムは対象環境を`environment_id`として参照しているプロジェクトが存在するかチェックしなければならない
- もし使用中のプロジェクトが1つ以上存在するならば、システムは削除を拒否し409 Conflictを返さなければならない
- エラーレスポンスには使用中のプロジェクト名一覧を含めなければならない

### AC-002: UIでの削除ボタン制御

- 環境管理画面において、プロジェクトで使用中の環境には削除ボタンを無効化しなければならない
- 無効化された削除ボタンには「X個のプロジェクトで使用中」というツールチップを表示しなければならない

### AC-003: 使用中でない環境

- どのプロジェクトにも使用されていない環境は、従来通り削除可能でなければならない
