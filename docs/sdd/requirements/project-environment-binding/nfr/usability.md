# ユーザビリティ要件

> 非機能要件カテゴリ: ユーザビリティ

## 概要

プロジェクトと環境の紐付けに関するUI操作が直感的で、ユーザーが迷わず操作できることを保証する。

## 要件一覧

### NFR-USA-001: 環境選択の直感性

**EARS記法**: プロジェクト作成画面において、システムは環境選択ドロップダウンに環境名とタイプを表示し、ユーザーが即座に識別できなければならない

**測定基準**:
- 指標: 環境選択に必要なクリック数
- 目標値: 2クリック以内（ドロップダウン展開 + 選択）
- 測定方法: UIフローの手動確認

**関連する機能要件**: FR-003

**優先度**: 中

### NFR-USA-002: 変更不可時のフィードバック

**EARS記法**: もしプロジェクトの環境変更が不可能な場合、システムは変更不可の理由を明確に表示しなければならない

**測定基準**:
- 指標: 理由表示の有無と明確さ
- 目標値: セッション数を含む具体的な理由メッセージ
- 測定方法: UIフローの手動確認

**関連する機能要件**: FR-011

**優先度**: 高

### NFR-USA-003: 削除制限の視覚的フィードバック

**EARS記法**: 環境管理画面において、使用中の環境は視覚的に区別可能でなければならない

**測定基準**:
- 指標: 使用中環境の識別可能性
- 目標値: 削除ボタンの無効化 + ツールチップでの使用プロジェクト数表示
- 測定方法: UIフローの手動確認

**関連する機能要件**: FR-008

**優先度**: 中
