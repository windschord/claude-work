# デフォルト環境概念の廃止 - 要件定義

## 概要

| 項目 | 内容 |
|------|------|
| 機能名 | デフォルト環境概念の廃止・環境選択必須化 |
| 目的 | 暗黙的なデフォルト環境の自動選択を廃止し、ユーザーが明示的に環境を選択する設計に変更する |
| 対象ユーザー | ClaudeWorkの全ユーザー |
| ステータス | DRAFT |

## 背景

現在、ExecutionEnvironmentテーブルに`is_default`フィールドがあり、サーバー起動時にデフォルト環境が自動作成される。プロジェクト/セッション作成時に環境が未指定の場合はデフォルト環境にフォールバックする設計となっている。しかし「デフォルト」ラベルの意味がユーザーに不明瞭であり、暗黙的な自動選択は意図しない環境での実行を招く恐れがある。

## ユーザーストーリー

- [US-001](stories/US-001.md): プロジェクト作成時の環境選択必須化
- [US-002](stories/US-002.md): セッション作成時の環境バリデーション
- [US-003](stories/US-003.md): デフォルト環境ラベルの廃止

## 非機能要件

- [後方互換性](nfr/compatibility.md): 既存データへの影響を最小限に抑える

## 機能要件（EARS記法）

### REQ-RDE-001: is_defaultフィールドの廃止

システムは、ExecutionEnvironmentテーブルから`is_default`フィールドを削除しなければならない。

**受入基準:**
- DBスキーマから`is_default`カラムが削除されている
- 既存の環境データ（host-default, docker-default等）は通常の環境として残る
- `is_default`を参照するすべてのコードが削除/修正されている

### REQ-RDE-002: サーバー起動時の自動環境作成の廃止

サーバー起動時、システムはデフォルト環境の自動作成を行ってはならない。

**受入基準:**
- `ensureDefaultExists()`メソッドが削除されている
- `ensureDefaultEnvironment()`メソッドが削除されている
- `DEFAULT_HOST_ENVIRONMENT`, `DEFAULT_DOCKER_ENVIRONMENT`定数が削除されている
- server.tsからデフォルト環境初期化の呼び出しが削除されている

### REQ-RDE-003: プロジェクト作成時の環境選択必須化

プロジェクト作成の時、システムは環境の選択を必須としなければならない。

**受入基準:**
- AddProjectModal: 環境未選択状態では作成ボタンが無効化されている
- AddProjectWizard: 環境未選択では次のステップに進めない
- RemoteRepoForm: 環境未選択では送信が無効化されている
- 環境が0件の場合、「環境が登録されていません。設定画面から環境を作成してください」というエラーメッセージが表示される

### REQ-RDE-004: セッション作成APIの環境必須バリデーション

セッション作成APIリクエストの時、もしプロジェクトに`environment_id`が設定されていないならば、システムはエラーレスポンスを返さなければならない。

**受入基準:**
- プロジェクトの`environment_id`がnullの場合、セッション作成APIが400エラーを返す
- デフォルト環境へのフォールバックロジックが削除されている
- `getDefault()`メソッドが削除されている

### REQ-RDE-005: UIからデフォルトラベルの削除

システムは、環境カードおよび環境セレクターから「デフォルト」ラベル表示を削除しなければならない。

**受入基準:**
- EnvironmentCard: 「デフォルト」バッジが表示されない
- StepEnvironment: 「(default)」テキストが表示されない
- 環境ソート順がis_defaultに依存しない

### REQ-RDE-006: 環境削除時のis_defaultチェック廃止と使用中チェック追加

環境削除の時、システムはis_defaultによる削除制限に代わり、セッションから参照されている環境の削除を禁止しなければならない。

**受入基準:**
- `is_default`による削除制限が削除されている
- セッションの`environment_id`で参照されている環境は削除不可
- プロジェクトの`environment_id`で参照されている環境は削除不可
- 参照元がない環境は削除可能
- 削除不可の場合、参照元の情報を含むエラーメッセージが表示される

### REQ-RDE-007: 環境インターフェースからis_defaultの削除

システムは、Environment型定義およびAPIレスポンスから`is_default`フィールドを削除しなければならない。

**受入基準:**
- `useEnvironments.ts`のEnvironmentインターフェースから`is_default`が削除されている
- 環境一覧APIレスポンスに`is_default`が含まれない
- すべてのテストのモックデータから`is_default`が削除されている
