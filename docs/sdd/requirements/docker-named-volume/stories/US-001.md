# US-001: Docker名前付きVolumeによるClaude Code設定の永続化

## ユーザーストーリー

**私は** ClaudeWorkの管理者として
**〜したい** Docker環境のClaude Code設定（~/.claude, ~/.config/claude）を名前付きVolumeで永続化したい
**なぜなら** Gitリポジトリの永続化方式（名前付きVolume）と統一し、Docker標準のVolume管理ツールで設定データを管理できるようにするため

## 受入基準（EARS記法）

### 機能要件

#### Volume作成

- **REQ-001-001**: Docker環境の作成時、システムは`claude-config-{env-id}`の命名規則で名前付きDocker Volumeを作成しなければならない
- **REQ-001-002**: 名前付きVolume作成時、システムはDockerode APIを使用してVolumeを作成しなければならない

#### コンテナマウント

- **REQ-001-003**: Dockerコンテナ起動時、システムは名前付きVolumeをコンテナ内`/home/node/.claude`にマウントしなければならない
- **REQ-001-004**: Dockerコンテナ起動時、システムは名前付きVolumeをコンテナ内`/home/node/.config/claude`にもマウントしなければならない

**採用方式**: 2つの名前付きVolume
  - `claude-config-claude-{env-id}` -> `/home/node/.claude`
  - `claude-config-configclaude-{env-id}` -> `/home/node/.config/claude`

#### 設定永続化

- **REQ-001-005**: コンテナ停止後に同じ環境IDで新しいコンテナを起動した時、システムは前回のClaude Code設定を保持した状態でコンテナを起動しなければならない
- **REQ-001-006**: `docker volume ls`コマンド実行時、作成されたVolumeが一覧に表示されなければならない

#### 環境削除

- **REQ-001-007**: Docker環境の削除時、システムは関連する名前付きVolumeを削除しなければならない
- **REQ-001-008**: もしVolume削除に失敗した場合、システムは警告をログに記録し、環境レコードの削除は続行しなければならない

#### バインドマウント廃止

- **REQ-001-009**: 新規作成されたDocker環境において、システムはホスト側の認証ディレクトリを作成してはならない
- **REQ-001-010**: もし環境に`auth_dir_path`が設定されていない場合、システムはバインドマウントの代わりに名前付きVolumeを使用しなければならない

#### 既存環境との互換性

- **REQ-001-011**: もし環境に`auth_dir_path`が既に設定されている場合、システムは従来通りバインドマウントを使用しなければならない

## データ要件

### Volume命名規則

| 用途 | Volume名パターン | 例 |
|------|-----------------|-----|
| Claude設定 | `claude-config-claude-{env-id}` | `claude-config-claude-abc12345` |
| Claude config設定 | `claude-config-configclaude-{env-id}` | `claude-config-configclaude-abc12345` |

### DB変更

- `auth_dir_path`カラムは引き続き使用するが、新規環境ではNULLのまま
- Volume名は環境IDから算出可能なため、DBに保存する必要はない
