# US-001: HOST側ポート使用状況チェック

## ユーザーストーリー

**私は** ClaudeWork管理者として
**〜したい** Docker環境のポートマッピング設定時に、HOST側ポートが使用中かどうかを事前に確認したい
**なぜなら** コンテナ起動時のバインドエラーを事前に防止し、設定の手戻りを減らしたいから

## 受入基準（EARS記法）

### 機能要件

- **REQ-001-001**: ポートチェックが要求された時、システムは`net.createServer()`を使用してHOSTポートへのバインドを試行し、使用可否を判定しなければならない
- **REQ-001-002**: ポートチェックが要求された時、システムはClaudeWork内の他の環境で設定済みのホストポートとの競合を検出しなければならない
- **REQ-001-003**: 複数のポートマッピングが設定されている場合、システムは全てのホストポートを一括でチェックしなければならない
- **REQ-001-004**: システムが各ポートをチェックする時、1ポートあたり500ミリ秒のタイムアウトを設定しなければならない
- **REQ-001-005**: もしポートチェック中にエラーが発生した場合（権限不足・タイムアウト等）、システムはそのポートのステータスを`unknown`として返さなければならない

## データ要件

### ポートチェック結果

```typescript
interface PortCheckResult {
  port: number;
  status: 'available' | 'in_use' | 'unknown';
  usedBy?: string;           // 使用中のプロセス/環境名
  source?: 'os' | 'claudework'; // 競合元がOSか他のClaudeWork環境か
}
```

## 備考

- well-knownポート（1-1023）はroot権限が必要な場合があり、チェック失敗時は`unknown`を返す
- ポートチェックは一時的なバインドであり、チェック後すぐに解放する
- 自己除外ロジック: 編集モード時に現在の環境IDを`excludeEnvironmentId`パラメータとして渡す
