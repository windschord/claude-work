# US-002: コンテナ側ポート使用状況チェック

## ユーザーストーリー

**私は** ClaudeWork管理者として
**〜したい** 実行中のコンテナ内でポートが使用中かどうかを確認したい
**なぜなら** コンテナ内部のポート競合も事前に把握したいから

## 受入基準（EARS記法）

### 機能要件

- **REQ-002-001**: コンテナが実行中の場合、システムは`docker exec`を使用してコンテナ内のポート使用状況を確認しなければならない
- **REQ-002-002**: もしコンテナが停止中または未作成の場合、システムはコンテナ側チェックをスキップし、ホスト側チェック結果のみを返さなければならない
- **REQ-002-003**: コンテナ側チェックはオプション扱いとし、もし`docker exec`が失敗した場合（コンテナ内にツールがない等）、システムはコンテナ側チェック結果を`unknown`として返さなければならない

## データ要件

### コンテナポートチェック結果

```typescript
interface ContainerPortCheckResult {
  containerPort: number;
  status: 'available' | 'in_use' | 'unknown';
  usedBy?: string;  // コンテナ内で使用しているプロセス
}
```

## 備考

- コンテナ内のチェックには`ss`または`netstat`コマンドを使用する（コンテナイメージに含まれていない場合がある）
- 新規環境作成時はコンテナが存在しないため、コンテナ側チェックは不要
- チェックの信頼性はコンテナイメージの内容に依存する
