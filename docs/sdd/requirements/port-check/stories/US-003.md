# US-003: ポート使用状況のUI表示

## ユーザーストーリー

**私は** ClaudeWork管理者として
**〜したい** 環境設定フォームでポートの使用状況を視覚的に確認したい
**なぜなら** ポートマッピング設定時に競合の有無をすぐに把握したいから

## 受入基準（EARS記法）

### 機能要件

- **REQ-003-001**: システムは`POST /api/environments/check-ports`エンドポイントを提供し、指定されたポートの使用状況を返さなければならない
- **REQ-003-002**: ポートチェック実行中、システムはローディングインジケータ（Loader2アイコン）を表示しなければならない
- **REQ-003-003**: ポートが使用可能の場合、システムは緑色のチェックアイコン（CheckCircle2）で「利用可能」と表示しなければならない
- **REQ-003-004**: ポートが使用中の場合、システムは赤色の警告アイコン（AlertCircle）で「使用中」と表示しなければならない
- **REQ-003-005**: ポートが未確認の場合、システムはグレーの疑問符アイコン（HelpCircle）で「チェック不可」と表示しなければならない

## 画面/UI要件

### ポート使用状況の表示

| 状態 | アイコン | 色 | Lucideアイコン |
|------|---------|-----|---------------|
| 利用可能 | チェックマーク | 緑 | CheckCircle2 |
| 使用中 | 警告アイコン | 赤 | AlertCircle |
| チェック不可 | 不明アイコン | グレー | HelpCircle |
| チェック中 | スピナー | - | Loader2 |

### APIリクエスト/レスポンス

```typescript
// POST /api/environments/check-ports
// Request
interface CheckPortsRequest {
  ports: number[];
  excludeEnvironmentId?: string;  // 編集中の環境ID（自己除外用）
}

// Response
interface CheckPortsResponse {
  results: PortCheckResult[];
}
```

## 備考

- チェックはユーザーの明示的なアクションで実行（自動チェックは行わない）
- フォーム送信はポート使用中の警告があっても可能とする（警告のみ、ブロックしない）
- チェック結果はフォームの状態としてのみ保持し、永続化しない
