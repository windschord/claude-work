# 非機能要件: ユーザビリティ（Usability）

## 概要

新規ユーザーが直感的にDocker環境とリモートリポジトリ機能を利用でき、既存ユーザーもスムーズに移行できるようにする。

## 要件

### 直感的なUI

- **NFR-045**: プロジェクト追加モーダルにおいて、システムは「ローカル」「リモート」タブを明確に区別して表示しなければならない
- **NFR-046**: 環境選択ドロップダウンにおいて、システムは各環境のタイプを視覚的に区別（バッジ）しなければならない
- **NFR-047**: リモートプロジェクトにおいて、システムは「Remote」バッジを目立つように表示しなければならない
- **NFR-048**: デフォルト環境において、システムは「デフォルト」ラベルを明確に表示しなければならない

### エラーメッセージ

- **NFR-049**: URLが無効な時、システムは「有効なGitリポジトリURL（git@... または https://...）を入力してください」と具体的に表示しなければならない
- **NFR-050**: SSH認証に失敗した時、システムは「SSH鍵の設定を確認してください」とトラブルシューティングガイドへのリンクを表示しなければならない
- **NFR-051**: Dockerが利用できない時、システムは「Dockerが起動していません。インストールガイドを確認してください」とリンク付きで表示しなければならない
- **NFR-052**: クローンに失敗した時、システムはGitの元のエラーメッセージを含めて表示しなければならない

### オンボーディング

- **NFR-053**: 初回起動時、システムはDocker環境の利点を説明するツールチップを表示してもよい
- **NFR-054**: 環境管理画面において、システムは各環境タイプの説明を表示しなければならない
- **NFR-055**: リモートタブにおいて、システムはURL入力のプレースホルダーに例（`git@github.com:user/repo.git`）を表示しなければならない

### フィードバック

- **NFR-056**: クローン処理中、システムは進捗を示すローディングスピナーを表示しなければならない
- **NFR-057**: プロジェクト登録成功時、システムは「プロジェクトを登録しました」と通知しなければならない
- **NFR-058**: 環境切替時、システムは切替完了を通知しなければならない
- **NFR-059**: git pull完了時、システムは「最新の状態に更新しました」または「更新はありませんでした」と明確に通知しなければならない

### アクセシビリティ

- **NFR-060**: 環境選択ドロップダウンにおいて、システムはキーボードナビゲーションをサポートしなければならない
- **NFR-061**: バッジの色において、システムは色覚異常者にも識別可能な色を使用しなければならない
- **NFR-062**: エラーメッセージにおいて、システムは赤色だけでなくアイコンも併用しなければならない

### ドキュメント

- **NFR-063**: システムはDocker環境のセットアップガイドを提供しなければならない
- **NFR-064**: システムはSSH鍵の設定ガイドを提供しなければならない
- **NFR-065**: システムはリモートリポジトリのクローン方法を説明するチュートリアルを提供しなければならない

## テスト基準

- 新規ユーザーがチュートリアルなしでリモートリポジトリを登録できること
- エラーメッセージが具体的で実行可能な解決策を含むこと
- UIがキーボード操作のみで完結できること
- バッジや色が色覚異常者にも識別可能であること
