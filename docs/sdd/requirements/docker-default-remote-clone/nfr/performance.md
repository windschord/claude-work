# 非機能要件: 性能（Performance）

## 概要

Docker環境やリモートリポジトリのクローン処理において、ユーザー体験を損なわない性能を維持する。

## 要件

### Docker環境の初期化

- **NFR-029**: Docker環境の初期化は10秒以内に完了しなければならない
- **NFR-030**: もしDocker環境の初期化が10秒を超える場合、システムは進捗表示をユーザーに示さなければならない
- **NFR-031**: Docker環境の初期化中、システムは他の操作をブロックしてはならない（非同期処理）

### クローン処理

- **NFR-032**: 小規模リポジトリ（100MB未満）のクローンは1分以内に完了しなければならない
- **NFR-033**: 大規模リポジトリ（1GB以上）のクローンにおいて、システムはタイムアウトを設けてはならない
- **NFR-034**: クローン処理中、システムは進捗表示（ローディングスピナー）をユーザーに示さなければならない
- **NFR-035**: クローン処理は非同期で実行され、ユーザーは他の操作を継続できなければならない

### Git操作

- **NFR-036**: ブランチ一覧の取得は5秒以内に完了しなければならない
- **NFR-037**: git pullは通常のCLI実行と同等の速度で動作しなければならない
- **NFR-038**: もしGit操作が30秒を超える場合、システムは進捗表示を更新しなければならない

### リソース管理

- **NFR-039**: Dockerコンテナのメモリ使用量は1セッションあたり512MB以内に抑えなければならない
- **NFR-040**: Dockerコンテナの起動時間は5秒以内でなければならない
- **NFR-041**: 同時に10セッションが実行されている時でも、システムは応答性を保たなければならない

### UI応答性

- **NFR-042**: 環境選択ドロップダウンの表示は100ms以内でなければならない
- **NFR-043**: プロジェクト一覧の更新は500ms以内に完了しなければならない
- **NFR-044**: セッション作成フォームの表示は200ms以内でなければならない

## テスト基準

- Docker環境の初期化時間を計測すること
- 各種リポジトリサイズでのクローン時間を計測すること
- Git操作のレスポンスタイムを計測すること
- 同時セッション実行時の負荷テストを実施すること
- UI操作の応答時間を計測すること
