# US-003: process.cwd()使用時のコメント規則を設定

## ユーザーストーリー

開発者として、子プロセスの`cwd`に`process.cwd()`を使う場合、その意図がコメントで明記されていることで、コードレビュー時にバグを検出しやすくしたい。

## 要件

### 機能要件

- REQ-007: CLAUDE.mdのプロセス実行テスト規則に、`process.cwd()`使用時のコメント義務を記載しなければならない
- REQ-008: 意図が書かれていない`cwd: process.cwd()`はコードレビューで指摘対象とする旨を記載しなければならない

### 例

NG:
```typescript
const result = spawnSync('cmd', args, {
  cwd: process.cwd(), // 意図不明
});
```

OK:
```typescript
// ユーザーの作業ディレクトリで実行する必要がある（相対パスのファイル操作のため）
const result = spawnSync('cmd', args, {
  cwd: process.cwd(),
});
```

## 受入基準

- [ ] CLAUDE.mdに`process.cwd()`使用時のコメント義務が記載されている
- [ ] NG例とOK例が記載されている
