# US-001: subprocessテストでcwd/envアサーションを必須化

## ユーザーストーリー

開発者として、`spawnSync`/`execSync`をモックするテストでは`cwd`と`env`のアサーションが含まれていることを確認したい。それにより、CWDに依存するバグをテストで検知できるようにする。

## 要件

### 機能要件

- REQ-001: `spawnSync`/`execSync`をモックするテストにおいて、`cwd`オプションのアサーションを含めなければならない
- REQ-002: `spawnSync`/`execSync`をモックするテストにおいて、`env`オプションが明示的に渡されている場合、そのアサーションを含めなければならない
- REQ-003: `expect.objectContaining()`を使用する場合、`cwd`を省略してはならない（最低限`expect.any(String)`で検証）

### 具体的な対象

`cli.ts`の以下のspawnSync呼び出しに対するテスト強化:
- `buildNext()`: `npm run build:next`の実行（cwd: projectRoot）
- `startDaemon()`: pm2 startの実行（cwd: projectRoot）
- `stopDaemon()`: pm2 stopの実行（cwd: projectRoot）
- `restartDaemon()`: pm2 restartの実行（cwd: projectRoot）
- `showStatus()`: pm2 statusの実行（cwd: projectRoot）

注: `git-service.ts`は統合テスト形式（実gitコマンド実行）のため対象外。`env-validation.ts`の`which`コマンドはcwd不要のため対象外。

## 受入基準

- [ ] `cli.ts`のspawnSync呼び出しに対するユニットテストが存在する（新規作成または既存強化）
- [ ] 各テストで`cwd`オプションが検証されている
- [ ] `env`オプションが渡されている箇所では`env`も検証されている
- [ ] テストがCIで全てパスする
