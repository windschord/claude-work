# 要件定義: Docker環境でのHOST機能無効化

## 概要

| 項目 | 内容 |
|------|------|
| フィーチャー名 | disable-host-in-docker |
| 作成日 | 2026-02-26 |
| ステータス | ACTIVE |

## 背景

ClaudeWorkアプリケーションがDockerコンテナ内で動作している場合、HOST環境（ホストマシン上で直接Claude Codeを実行する機能）はコンテナ内部で実行されるため意味をなさない。さらに、コンテナ内でHOST環境を使用すると、意図しない動作やセキュリティリスクが生じる可能性がある。

現状ではDockerコンテナ内で動作していてもHOST環境がデフォルトで作成・利用可能になっており、ユーザーが誤ってHOST環境を選択してしまう問題がある。

## スコープ

### 対象

- Dockerコンテナ内動作の自動検出ロジック
- HOST環境の作成・利用制限
- デフォルト環境のDOCKERへの切り替え
- `ALLOW_HOST_ENVIRONMENT` 環境変数による制限解除
- 環境API・UIへの制限反映

### 対象外

- SSH環境の制限（今回のスコープ外）
- Docker環境自体の機能変更
- 既存のDocker環境作成・管理フロー

## EARS記法 要件一覧

### 機能要件

| ID | EARS記法 |
|----|---------|
| FR-001 | アプリケーションがDockerコンテナ内で動作している時、システムはHOST環境の新規作成を拒否しなければならない |
| FR-002 | アプリケーションがDockerコンテナ内で動作している時、システムは既存のHOST環境をセッション作成時に利用不可としなければならない |
| FR-003 | アプリケーションがDockerコンテナ内で動作している時、システムはデフォルト環境としてDOCKER環境を使用しなければならない |
| FR-004 | `/.dockerenv` ファイルが存在する場合、システムはDockerコンテナ内で動作していると判定しなければならない |
| FR-005 | 環境変数 `RUNNING_IN_DOCKER=true` が設定されている場合、システムはDockerコンテナ内で動作していると判定しなければならない |
| FR-006 | FR-004またはFR-005のいずれかが真の場合、システムはDockerコンテナ内と判定しなければならない（OR条件） |
| FR-007 | 環境変数 `ALLOW_HOST_ENVIRONMENT=true` が設定されている場合、システムはDockerコンテナ内でもHOST環境の作成・利用を許可しなければならない |
| FR-008 | HOST環境が無効化されている時、環境作成APIはHOSTタイプの作成リクエストに対してエラーレスポンス（403）を返さなければならない |
| FR-009 | HOST環境が無効化されている時、環境一覧APIはHOST環境に `disabled: true` フラグを含めて返さなければならない |
| FR-010 | HOST環境が無効化されている時、UIは環境作成フォームでHOSTタイプを選択不可にしなければならない |
| FR-011 | HOST環境が無効化されている時、UIは既存HOST環境カードに無効化状態を表示しなければならない |
| FR-012 | `docker-compose.yml` は `RUNNING_IN_DOCKER=true` を環境変数として含まなければならない |
| FR-013 | HOST環境が無効化されている時、セッション作成モーダルの実行環境選択リストからdisabled=trueのHOST環境を除外しなければならない |
| FR-014 | HOST環境が無効化されている時、プロジェクト環境設定で「Host (自動選択)」の代わりに「Docker (自動選択)」を表示しなければならない |
| FR-015 | HOST環境が無効化されている時、環境設定画面の環境カード一覧からdisabled=trueのHOST環境を非表示にしなければならない |
| FR-016 | HOST環境が無効化されている時、環境作成フォームのタイプ選択ドロップダウンからHOSTオプションを除外しなければならない |
| FR-017 | HOST環境が無効化されている時、プロジェクト追加モーダルではタブUIを表示せず、リモートリポジトリ用フォームのみを表示しなければならない |
| FR-018 | HOST環境が無効化されている時、RemoteRepoFormの保存場所選択から「ホスト環境」ラジオボタンを非表示にし、cloneLocationを'docker'に固定しなければならない |

### 非機能要件サマリー

| ID | EARS記法 |
|----|---------|
| NFR-001 | Docker内動作の検出は、サーバー起動時に1回だけ実行され、結果をキャッシュしなければならない |
| NFR-002 | HOST環境制限はAPIレベルで強制されなければならない（UIバイパス対策） |
| NFR-003 | ホスト環境で直接動作している場合（非Docker）、既存の動作に一切影響を与えてはならない |
