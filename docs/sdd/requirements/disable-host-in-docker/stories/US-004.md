# US-004: Docker環境でのHOST関連UI完全非表示

## ユーザーストーリー

**私は** ClaudeWorkのユーザーとして
**~したい** Docker環境で動作している時にHOST関連の選択肢やカードがUIに表示されない
**なぜなら** 利用不可のHOST環境が表示されると混乱を招き、誤操作の原因になるため

## 背景

PR#161でバックエンドのHOST機能無効化は完了しているが、UIレイヤーに以下の修正漏れがある:

1. セッション作成モーダルでdisabledなHOST環境が選択可能
2. プロジェクト環境設定で「Host (自動選択)」が表示される
3. 環境設定画面でdisabledなHOST環境がグレーアウト表示される（非表示にすべき）

## 受入基準（EARS記法）

### AC-001: セッション作成モーダルでのHOST環境フィルタリング

- HOST環境が無効化されている時、セッション作成モーダルの実行環境選択リストからdisabled=trueのHOST環境を除外しなければならない
- HOST環境が無効化されていない時（非Docker環境またはALLOW_HOST_ENVIRONMENT=true）、セッション作成モーダルにHOST環境を表示しなければならない

### AC-002: プロジェクト環境設定でのHOST表示制御

- HOST環境が無効化されている時、プロジェクト環境設定コンポーネントで「Host (自動選択)」の代わりに「Docker (自動選択)」を表示しなければならない
- HOST環境が無効化されていない時、従来通り「Host (自動選択)」を表示しなければならない

### AC-003: 環境設定画面でのHOST環境カード非表示

- HOST環境が無効化されている時、環境設定画面の環境カード一覧からdisabled=trueのHOST環境を非表示にしなければならない
- HOST環境が無効化されていない時、HOST環境カードを従来通り表示しなければならない

### AC-004: 環境作成フォームでのHOSTオプション非表示

- HOST環境が無効化されている時、環境作成フォームのタイプ選択ドロップダウンからHOSTオプションを除外しなければならない
- HOST環境が無効化されていない時、HOSTオプションを従来通り表示しなければならない
