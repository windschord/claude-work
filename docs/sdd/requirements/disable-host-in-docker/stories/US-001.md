# US-001: Docker内でのHOST環境自動無効化

## ユーザーストーリー

**私は** ClaudeWorkの管理者として
**〜したい** Dockerコンテナ内でアプリが動作している場合にHOST環境が自動的に無効化される
**なぜなら** コンテナ内でのHOST環境の使用は意図しない動作を引き起こし、ユーザーの混乱を招くため

## 受入基準（EARS記法）

### AC-001: Dockerコンテナ検出

- `/.dockerenv` ファイルが存在する時、システムはDockerコンテナ内で動作していると判定しなければならない
- 環境変数 `RUNNING_IN_DOCKER=true` が設定されている時、システムはDockerコンテナ内で動作していると判定しなければならない
- いずれかの条件が真の場合、システムはDockerコンテナ内と判定しなければならない

### AC-002: HOST環境の作成制限

- Dockerコンテナ内で動作している時、環境作成API（POST `/api/environments`）でtype=HOSTのリクエストは403エラーを返さなければならない
- エラーレスポンスには制限理由を示すメッセージを含めなければならない

### AC-003: HOST環境の利用制限

- Dockerコンテナ内で動作している時、HOST環境を指定したセッション作成は拒否されなければならない
- 既存のHOST環境は環境一覧に `disabled: true` フラグ付きで表示されなければならない

### AC-004: UI制限

- Dockerコンテナ内で動作している時、環境作成フォームでHOSTタイプは選択不可（disabled）でなければならない
- 既存のHOST環境カードには無効化を示す視覚的インジケーターを表示しなければならない
