# US-005: プロジェクト追加画面でのローカル選択肢制御

## ユーザーストーリー

**私は** ClaudeWorkのユーザーとして
**したい** HOST環境が利用不可の場合にプロジェクト追加画面でローカル関連の選択肢が表示されない
**なぜなら** 利用できないローカル追加やホストClone操作を選択してしまい、エラーや混乱を招くことを防ぎたいため

## 背景

プロジェクト追加モーダル（AddProjectModal）には以下の2つの経路がある:

1. **ローカルタブ**: ホストマシン上の既存Gitリポジトリパスを指定して追加（`clone_location='host'`固定）
2. **リモートタブ**: リモートURLからClone（RemoteRepoForm内の保存場所選択で`docker`/`host`を選択可能）

HOST環境が利用不可（`hostEnvironmentDisabled=true`）の場合、ローカルタブの機能はホスト環境でのみ動作するため意味をなさない。同様に、リモートタブの保存場所「ホスト環境」もホスト上でのClone操作となるため利用不可である。

既存のUS-004ではセッション作成モーダル、プロジェクト環境設定、環境設定画面でのHOST関連UI非表示を実装済み。本ストーリーはプロジェクト追加画面の修正漏れを補完する。

## 受入基準（EARS記法）

### AC-001: ローカルタブの非表示

- HOST環境が無効化されている時、プロジェクト追加モーダルの「ローカル」タブを非表示にしなければならない
- HOST環境が無効化されている時、プロジェクト追加モーダルはタブUIを表示せずリモートフォームのみを表示しなければならない
- HOST環境が無効化されていない時（非Docker環境またはALLOW_HOST_ENVIRONMENT=true）、プロジェクト追加モーダルは従来通り「ローカル」タブと「リモート」タブの両方を表示しなければならない

### AC-002: リモートタブ内の保存場所「ホスト環境」の非表示

- HOST環境が無効化されている時、RemoteRepoFormの保存場所選択から「ホスト環境」ラジオボタンを非表示にしなければならない
- HOST環境が無効化されている時、RemoteRepoFormの`cloneLocation`は`'docker'`に固定されなければならない
- HOST環境が無効化されている時、保存場所セクション自体を非表示にしなければならない（Docker環境のみが残り、選択の余地がないため）
- HOST環境が無効化されていない時、従来通り「Docker環境」と「ホスト環境」のラジオボタンを両方表示しなければならない

### AC-003: タブ非表示時の直接フォーム表示

- HOST環境が無効化されてローカルタブが非表示の場合、Tab.Groupを描画せずRemoteRepoFormを直接表示しなければならない（タブUI自体がスキップされるため、defaultIndex調整は不要）

### AC-004: hostEnvironmentDisabledフラグの活用

- AddProjectModalコンポーネントは`useEnvironments()`フックから`hostEnvironmentDisabled`フラグを取得して制御に使用しなければならない
- RemoteRepoFormコンポーネントは`useEnvironments()`フックから`hostEnvironmentDisabled`フラグを取得するか、propsとして受け取って制御に使用しなければならない

## 影響範囲

### 変更対象コンポーネント

| ファイル | 変更内容 |
|---------|---------|
| `src/components/projects/AddProjectModal.tsx` | ローカルタブの条件付き非表示、Tab.Groupスキップによる直接フォーム表示 |
| `src/components/projects/RemoteRepoForm.tsx` | 保存場所ラジオボタンの条件付き非表示、cloneLocation固定 |

### 変更不要なコンポーネント

- バックエンドAPI（既に`hostEnvironmentDisabled`フラグを返している）
- `useEnvironments()`フック（既に`hostEnvironmentDisabled`を返している）
- データベーススキーマ
