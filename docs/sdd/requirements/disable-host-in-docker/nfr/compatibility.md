# NFR: 互換性

## 要件

| ID | EARS記法 |
|----|---------|
| NFR-COMPAT-001 | ホスト環境で直接動作している場合（非Docker）、既存の動作に一切影響を与えてはならない |
| NFR-COMPAT-002 | Docker内動作の検出はサーバー起動時に1回だけ実行され、結果をキャッシュしなければならない |
| NFR-COMPAT-003 | 既存のDOCKER環境の作成・管理フローは変更されてはならない |
| NFR-COMPAT-004 | 既存のAPIレスポンス形式は、`disabled` フィールドの追加を除いて変更されてはならない |
