# US-004: Registry Firewall管理UIへのアクセス

## ユーザーストーリー

**私は** ClaudeWorkの管理者として
**~したい** registry-firewallの管理UIにClaudeWorkからアクセスしたい
**なぜなら** ブロックログの確認やカスタムルールの管理をシームレスに行いたいため

## 受入基準(EARS記法)

### 機能要件

- **REQ-009**: 環境設定画面において、システムはregistry-firewallの管理UIへのリンクを提供しなければならない
  - リンク先: registry-firewallの管理UI (`/ui`)
  - ClaudeWorkのAPIを経由してプロキシすることで、外部ポート公開なしでアクセス可能にする
  - プロキシパス: `/api/registry-firewall/ui/*` -> registry-firewallの`/ui/*`

- **REQ-010**: 環境設定画面において、システムはregistry-firewallの最近のブロックログを表示しなければならない
  - registry-firewallの`/api/blocks`エンドポイントから取得
  - 最新10件を表示
  - 表示項目: 日時、パッケージ名、レジストリ種別、ブロック理由
  - 「全てのログを見る」リンクでregistry-firewall管理UIに遷移

## 画面/UI要件

### ブロックログ表示

```text
+-------------------------------------------+
| 最近のブロックログ                          |
+-------------------------------------------+
| 2026-03-15 10:30  npm   evil-pkg   OSV    |
| 2026-03-15 09:15  pypi  malware    OpenSSF|
| ...                                        |
| [全てのログを見る →]                        |
+-------------------------------------------+
```

## データ要件

### APIエンドポイント

- `GET /api/registry-firewall/blocks` - ブロックログ取得
  - registry-firewallの`/api/blocks`にプロキシ
  - クエリパラメータ: `limit` (デフォルト: 10)
  - 認証: registry-firewallのAPIトークンをサーバー側で管理

- `GET /api/registry-firewall/ui/*` - 管理UIプロキシ
  - registry-firewallの`/ui/*`へリバースプロキシ
  - 静的ファイル(JS, CSS, HTML)を含む

## 備考

- registry-firewallのAPI認証トークンは環境変数(`REGISTRY_FIREWALL_API_TOKEN`)で設定
- UIプロキシはNext.jsのrewrites機能またはサーバーサイドプロキシで実装
- registry-firewallのUIは独立したReactアプリケーション。iframeでの埋め込みも検討可能だが、まずはリンク遷移を推奨
