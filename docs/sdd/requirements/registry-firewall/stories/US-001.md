# US-001: Docker Compose統合

## ユーザーストーリー

**私は** ClaudeWorkの管理者として
**~したい** registry-firewallをDocker Compose構成に組み込みたい
**なぜなら** Docker実行環境でパッケージレベルのサプライチェーン攻撃保護を提供するため

## 受入基準(EARS記法)

### 機能要件

- **REQ-001**: `docker compose up`の時、システムはregistry-firewallコンテナを起動しなければならない
  - イメージ: `ghcr.io/windschord/registry-firewall:latest`
  - ポート: 8080(内部通信用、ホストへの公開は不要)
  - ヘルスチェック: `/health`エンドポイントで確認
  - 依存関係: appサービスはregistry-firewallの起動(service_started)に依存(NFR-AVA-001: healthy待ちをしない)

- **REQ-002**: registry-firewallコンテナ起動時、システムはデフォルト設定ファイル(config.yaml)を提供しなければならない
  - 対応レジストリ: npm(registry.npmjs.org), PyPI(pypi.org), Go(proxy.golang.org), Cargo(index.crates.io), Docker(registry-1.docker.io)
  - OSV/OpenSSFフィルタリング: 有効
  - キャッシュ: ファイルシステムベース、永続化ボリューム使用
  - データベース: SQLite、永続化ボリューム使用
  - 設定はDockerボリュームまたはbind mountで提供

- **REQ-003**: システムはregistry-firewallをclaudework-filterネットワークに接続しなければならない
  - Docker実行環境のコンテナからregistry-firewallにアクセス可能にする
  - defaultネットワークにも接続(appサービスとの通信用)

## データ要件

### Docker Compose追加サービス定義

```yaml
registry-firewall:
  image: ghcr.io/windschord/registry-firewall:latest
  container_name: claudework-registry-firewall
  networks:
    - default
    - claudework-filter
  volumes:
    - registry-firewall-data:/data
    - ./configs/registry-firewall.yaml:/config/config.yaml:ro
  environment:
    - REGISTRY_FIREWALL_CONFIG=/config/config.yaml
    - RUST_LOG=${REGISTRY_FIREWALL_LOG_LEVEL:-info}
  restart: unless-stopped
  healthcheck:
    test: ["CMD", "wget", "--spider", "-q", "http://localhost:8080/health"]
    interval: 30s
    timeout: 5s
    retries: 3
```

### 追加ボリューム

```yaml
volumes:
  registry-firewall-data:
    driver: local
```

## 備考

- registry-firewallのDocker imageはghcr.ioに公開予定。公開されるまではDocker Compose起動時にエラーとなる
- npm対応は registry-firewall Issue #23 で対応予定。設定ファイルにはnpmを含めるが、対応前はnpmプロキシ経由のリクエストが失敗する可能性がある
- 監視スタック(OTel, Jaeger, Prometheus)は含めない。必要な場合は別途追加
