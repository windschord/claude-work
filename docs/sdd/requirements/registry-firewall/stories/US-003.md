# US-003: UI管理画面(有効/無効・ステータス表示)

## ユーザーストーリー

**私は** ClaudeWorkの管理者として
**~したい** registry-firewallの有効/無効をUIから切り替え、ステータスを確認したい
**なぜなら** セキュリティ機能の管理を直感的に行いたいため

## 受入基準(EARS記法)

### 機能要件

- **REQ-007**: 設定画面において、システムはregistry-firewallの有効/無効をトグルスイッチで切り替えられなければならない
  - 設定場所: 環境設定ページ(`/settings/environments`)の全体設定セクション
  - トグル変更時、システムは設定をサーバーに保存しなければならない
  - もしregistry-firewallが無効の場合、新規コンテナ起動時にレジストリプロキシ設定を注入しない

- **REQ-008**: 環境設定画面において、システムはregistry-firewallのヘルスステータスを表示しなければならない
  - ステータス表示: 正常(green)/異常(red)/不明(gray)
  - registry-firewallの`/health`エンドポイントでステータスを取得
  - もしregistry-firewallコンテナが停止している場合、システムは「停止中」と表示しなければならない

## 画面/UI要件

### 環境設定ページへの追加

既存の`/settings/environments`ページに「パッケージセキュリティ」セクションを追加:

```
+-------------------------------------------+
| パッケージセキュリティ                       |
+-------------------------------------------+
| Registry Firewall                          |
| ステータス: [●正常] / [●異常] / [●停止中]    |
|                                            |
| 有効/無効: [トグルスイッチ]                   |
|                                            |
| 対応レジストリ: npm, PyPI, Go, Cargo, Docker |
|                                            |
| [管理画面を開く →]                           |
+-------------------------------------------+
```

## データ要件

### 設定保存

グローバル設定として保存:
- 保存先: 設定API (`/api/settings/config`) の一部として管理
- キー: `registryFirewall.enabled` (boolean)

### ヘルスチェックAPI

新規エンドポイント:
- `GET /api/registry-firewall/health` - registry-firewallのヘルスステータスを返す
  - registry-firewallの`/health`エンドポイントにプロキシ
  - レスポンス: `{ status: "healthy" | "unhealthy" | "stopped", registries: [...] }`

## 備考

- 初期状態はregistry-firewallが有効(デフォルト値)
- 有効/無効の切替は、既に実行中のコンテナには影響しない(新規起動時のみ適用)
- registry-firewallサービス自体の起動/停止は制御しない(Docker Composeで常時起動)
