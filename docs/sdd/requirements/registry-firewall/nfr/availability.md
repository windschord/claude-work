# 可用性要件

> 非機能要件カテゴリ: 可用性

## 概要

registry-firewallの障害がDocker実行環境の利用に致命的な影響を与えないことを保証する。

## 要件一覧

### NFR-AVA-001: registry-firewall障害時のフォールバック

**EARS記法**: もしregistry-firewallが停止または応答しない場合、Docker実行環境のコンテナは起動可能でなければならない

**測定基準**:
- 指標: registry-firewall停止時のコンテナ起動成功率
- 目標値: 100%
- 測定方法: registry-firewallコンテナを停止した状態でDocker実行環境を作成

**関連する機能要件**: REQ-004, REQ-005

**優先度**: 高

**検証方法**:
- [ ] registry-firewallコンテナ停止時にDocker実行環境の起動が成功することを確認
- [ ] レジストリプロキシ設定は注入されるが、パッケージインストール失敗時のエラーメッセージが理解可能であることを確認
- [ ] UIでregistry-firewallの停止状態が表示されることを確認

---

### NFR-AVA-002: ヘルスチェック応答性

**EARS記法**: registry-firewallは`/health`エンドポイントに対して2秒以内にレスポンスを返さなければならない

**測定基準**:
- 指標: ヘルスチェック応答時間
- 目標値: 2秒以内
- 測定方法: `/health`エンドポイントへのHTTPリクエスト応答時間を計測

**関連する機能要件**: REQ-008

**優先度**: 中

**検証方法**:
- [ ] ヘルスチェックAPIが2秒以内に応答することを確認
- [ ] タイムアウト時にUI上で「停止中」ステータスが表示されることを確認
