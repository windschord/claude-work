# セキュリティ要件

> 非機能要件カテゴリ: セキュリティ

## 概要

registry-firewall統合におけるセキュリティ要件。パッケージレジストリプロキシの安全な運用と、既存のネットワークフィルタリングとの整合性を確保する。

## 要件一覧

### NFR-SEC-001: registry-firewallのAPI認証

**EARS記法**: システムはregistry-firewallのAPIエンドポイントにアクセスする際、認証トークンを使用しなければならない

**測定基準**:
- 指標: 認証なしでのAPI呼び出し成功率
- 目標値: 0%(全てのAPI呼び出しが認証済み)
- 測定方法: registry-firewallのアクセスログで認証なしリクエストを確認

**関連する機能要件**: REQ-009, REQ-010

**優先度**: 高

**検証方法**:
- [ ] API呼び出し時にAuthorizationヘッダーが含まれることを確認
- [ ] 認証トークンがサーバー側で管理され、フロントエンドに露出しないことを確認

---

### NFR-SEC-002: コンテナからのregistry-firewall管理API非アクセス

**EARS記法**: Docker実行環境のコンテナからは、registry-firewallの管理APIエンドポイント(/api/*)にアクセスできてはならない

**測定基準**:
- 指標: コンテナからの管理APIアクセス成功率
- 目標値: 0%
- 測定方法: コンテナ内からcurlで管理APIにアクセスし、失敗することを確認

**関連する機能要件**: REQ-003, REQ-006

**優先度**: 高

**検証方法**:
- [ ] コンテナ内から`curl http://claudework-registry-firewall:8080/api/blocks`が認証エラーとなることを確認
- [ ] registry-firewallの認証設定が有効であることを確認
- [ ] コンテナにはAPIトークンが注入されないことを確認

---

### NFR-SEC-003: レジストリプロキシ設定の安全性

**EARS記法**: システムはregistry-firewallへの接続情報をコンテナ環境変数に限定して注入しなければならない

**測定基準**:
- 指標: API認証トークンのコンテナへの露出
- 目標値: 0件
- 測定方法: コンテナの環境変数とファイルシステムを検査

**関連する機能要件**: REQ-004, REQ-005

**優先度**: 高

**検証方法**:
- [ ] コンテナ環境変数にAPIトークンが含まれないことを確認
- [ ] コンテナに注入される情報はレジストリURL(HTTP)のみであることを確認
