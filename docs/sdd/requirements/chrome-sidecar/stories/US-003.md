# US-003: 環境設定でのサイドカー構成

## ユーザーストーリー

**私は** claude-workの利用者として
**〜したい** 環境設定画面でサイドカーChromeの有効/無効やイメージを設定できるようにしたい
**なぜなら** プロジェクトや用途に応じてブラウザ検証環境の使用を選択したいから

## 受入基準（EARS記法）

### 機能要件

- **REQ-003-001**: Docker環境の設定画面において、システムはサイドカーChromeの有効/無効トグルを提供しなければならない
- **REQ-003-002**: もしサイドカーが有効に設定されているならば、システムはChromeイメージ名とタグのカスタマイズフィールドを表示しなければならない
- **REQ-003-003**: サイドカー設定が未指定またはデフォルトの場合、システムはPR#186の単一コンテナ方式（chrome-devtools拡張イメージ）をデフォルト動作として維持しなければならない

## UI要件

### 環境設定画面（/settings/environments）の拡張

環境編集フォームに以下を追加:

| フィールド | 型 | デフォルト | 説明 |
|-----------|------|-----------|------|
| Chrome Sidecar | トグル | OFF | サイドカーChrome有効/無効 |
| Chrome Image | テキスト | chromium/headless-shell | Chromeコンテナイメージ名 |
| Chrome Tag | テキスト | （固定バージョン指定） | Chromeイメージタグ。再現性のため固定バージョンタグを推奨 |

- サイドカーがOFFの場合、Chrome Image/Tagフィールドは非活性（disabled）
- Docker環境でのみ表示（HOST環境では非表示）

## データ要件

### ExecutionEnvironment.config JSON拡張

```json
{
  "imageName": "claude-code-sandboxed",
  "imageTag": "latest",
  "chromeSidecar": {
    "enabled": false,
    "image": "chromium/headless-shell",
    "tag": "131.0.6778.204"
  }
}
```

## 備考

- 既存のDocker環境設定との後方互換性を維持（chromeSidecarキーが存在しない場合はOFFとして扱う）
- config JSONのスキーマバリデーションを追加
- Chromeイメージタグは再現性と障害切り分けのため固定バージョン（例: `131.0.6778.204`）を推奨。`latest` の使用は非推奨とし、明示的な更新運用を前提とする
