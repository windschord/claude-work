# US-002: サイドカーChromeのライフサイクル管理

## ユーザーストーリー

**私は** claude-workの管理者として
**〜したい** セッション終了時にChromeサイドカーコンテナとネットワークが自動的にクリーンアップされるようにしたい
**なぜなら** リソースリークを防止し、システムの安定性を維持したいから

## 受入基準（EARS記法）

### 機能要件

- **REQ-002-001**: サイドカー付きセッションが破棄された時、システムはClaude Codeコンテナ停止後にChromeサイドカーコンテナを停止・削除し、セッション専用ネットワークを削除しなければならない
- **REQ-002-002**: サーバー起動時、システムはDBに記録されたchrome_container_idに対応するコンテナが実行中でない場合、そのコンテナを削除し、関連するセッション専用ネットワーク（`cw-net-*`）も削除しなければならない
- **REQ-002-003**: セッションのライフサイクルイベント（作成・破棄）の時、システムはSessionテーブルのchrome_container_idとchrome_debug_portを適切に更新しなければならない

## 関連ストーリー

## 備考

- 破棄順序: Claude停止 → Chrome停止 → ネットワーク削除（依存関係順）
- AutoRemove: trueを設定するため、コンテナ停止と同時に自動削除される
- ネットワーク削除はコンテナが全て切断された後に実行
- 孤立ネットワーク検出: `cw-net-` プレフィックスで識別
