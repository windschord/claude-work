# US-004: ブラウザデバッグアクセス

## ユーザーストーリー

**私は** claude-workの開発者として
**〜したい** 各セッションのChromeサイドカーにホストブラウザからアクセスしてデバッグできるようにしたい
**なぜなら** Claude Codeがブラウザ上で行っている操作を目視確認し、問題をデバッグしたいから

## 受入基準（EARS記法）

### 機能要件

- **REQ-004-001**: Chromeサイドカーコンテナ起動時、システムはCDP（Chrome DevTools Protocol）ポート9222をホスト側の空きポートに動的にマッピングしなければならない
- **REQ-004-002**: セッション詳細画面において、もしサイドカーChromeが起動中ならば、システムはデバッグ用のホストポート番号を表示しなければならない

## UI要件

### セッション情報表示

セッション詳細またはセッションリストに以下を表示:

- Chrome Debugポート: `localhost:<mapped-port>` 形式
- Chromeステータス: 起動中/停止中

## 備考

- ホスト側ポートはDocker APIの動的割当（HostPort: ''）で自動決定
- `chrome://inspect` をホストブラウザで開き、`localhost:<port>` で接続可能
- デバッグポートはSessionテーブルのchrome_debug_portカラムに永続化
