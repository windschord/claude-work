# US-004: ブラウザデバッグアクセス

## ユーザーストーリー

**私は** claude-workの開発者として
**〜したい** 各セッションのChromeサイドカーにホストブラウザからアクセスしてデバッグできるようにしたい
**なぜなら** Claude Codeがブラウザ上で行っている操作を目視確認し、問題をデバッグしたいから

## 受入基準（EARS記法）

### 機能要件

- **REQ-004-001**: Chromeサイドカーコンテナ起動時、システムはCDP（Chrome DevTools Protocol）ポート9222をホスト側の空きポートに動的にマッピングしなければならない。ホスト側バインドアドレスは `127.0.0.1` に固定し、外部ネットワークからのアクセスを防止しなければならない。ポートマッピングに失敗した場合は、サイドカーChromeをホストポートなしで起動し（コンテナ内部のCDP通信は維持）、警告レベルのログを出力しなければならない
- **REQ-004-002**: セッション詳細画面において、もしサイドカーChromeが起動中ならば、システムはデバッグ用のホストポート番号を表示しなければならない
- **REQ-004-003**: Chrome Debugポートとステータスの表示は、セッション所有者または管理者権限を持つユーザーのみに許可しなければならない。セッション詳細画面とセッションリスト画面の両方で、表示前にユーザーの権限を検証しなければならない

## UI要件

### セッション情報表示

セッション詳細またはセッションリストに以下を表示:

- Chrome Debugポート: `localhost:<mapped-port>` 形式
- Chromeステータス: 起動中/停止中

## データ要件

### Sessionテーブル拡張

| カラム名 | 型 | NULL可否 | デフォルト | 説明 |
|---------|------|----------|-----------|------|
| chrome_debug_port | INTEGER | YES | NULL | サイドカーChrome起動時にマッピングされたホスト側ポート番号。サイドカー未使用時はNULL |

- ポート割当成功時のみ設定（REQ-004-001のdegraded modeではNULLのまま）、停止時にNULLにクリア
- セッション削除時にカスケード削除（Sessionレコードごと削除）

## 備考

- ホスト側ポートはDocker APIの動的割当（HostPort: '', HostIp: '127.0.0.1'）で自動決定。Dockerのデフォルトエフェメラルポート範囲（通常32768-60999）に従う
- ポート割当失敗時（エフェメラルポート枯渇等）: セッション作成自体は続行し、サイドカーChromeはデバッグポートなし（chrome_debug_port=NULL）で起動する。エラーはログに記録し、UIでデバッグポート未割当として表示する
- `chrome://inspect` をホストブラウザで開き、`localhost:<port>` で接続可能（ローカルホストのみ）
- デバッグポートはSessionテーブルのchrome_debug_portカラムに永続化（データ要件セクション参照）
