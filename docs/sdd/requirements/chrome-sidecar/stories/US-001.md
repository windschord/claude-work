# US-001: サイドカーChromeの自動起動・接続

## ユーザーストーリー

**私は** claude-workの利用者として
**〜したい** Dockerセッション作成時に独立したChromeコンテナが自動起動し、Claude CodeがChrome DevTools MCPで接続できるようにしたい
**なぜなら** セッションごとに独立したブラウザ検証環境が必要であり、手動でChromeを起動・設定する手間を省きたいから

## 受入基準（EARS記法）

### 機能要件

- **REQ-001-001**: サイドカーが有効な環境でDockerセッションが作成された時、システムはセッション専用のDockerブリッジネットワークを作成し、chromium公式イメージのChromeコンテナを起動しなければならない
- **REQ-001-002**: Chromeサイドカーコンテナが起動した時、システムはClaude Codeコンテナを同じDockerネットワークに接続し、Chrome DevTools Protocol（CDP）経由で通信可能にしなければならない
- **REQ-001-003**: サイドカーが有効なセッション作成時、システムはClaude Codeコンテナの`.mcp.json`に`--browserUrl=http://<chrome-container-name>:9222`を含むchrome-devtools MCP設定を動的に生成・注入しなければならない
- **REQ-001-004**: Chromeサイドカーコンテナ起動後、システムはCDP（ポート9222）のヘルスチェックが成功するまで待機してからClaude Codeコンテナを起動しなければならない

## データ要件

### Sessionテーブル拡張

| カラム | 型 | 説明 |
|--------|------|------|
| chrome_container_id | TEXT | Chromeサイドカーコンテナ名（NULLの場合サイドカー未使用） |
| chrome_debug_port | INTEGER | ホストにマッピングされたデバッグポート番号（NULLの場合未マッピング） |

## 備考

- Chromeコンテナ名は `cw-chrome-<session-id>` 形式を使用
- ネットワーク名は `cw-net-<session-id>` 形式を使用
- Chrome Ready Checkのタイムアウトは最大30秒（DockerAdapterの既存パターンに合わせる）
