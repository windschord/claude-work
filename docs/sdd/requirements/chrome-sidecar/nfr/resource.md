# 非機能要件 - リソース

## NFR-RES-001: Chromeコンテナのメモリ制限

Chromeサイドカーコンテナの実行中、システムはコンテナのメモリ使用量を512MBに制限しなければならない。

**理由**: Chrome/Chromiumはメモリ消費が大きく（300-500MB/インスタンス）、無制限に使用するとホストの安定性に影響する。

## NFR-RES-002: 起動タイムアウト

Chromeサイドカーコンテナ起動時、システムはCDPヘルスチェックが30秒以内に成功しない場合、サイドカーなしでセッションを起動しなければならない。

**理由**: Chrome起動の遅延がセッション作成全体をブロックすることを防止する。

## NFR-RES-003: 同時サイドカー上限

システムは同時に起動するChromeサイドカーコンテナ数を監視可能にし、ログに記録しなければならない。

**理由**: リソース消費の把握と、将来的な上限制御の基盤とする。
