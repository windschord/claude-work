# US-001: DockerボリュームID参照によるWorktree作成

## 関連Issue

- GitHub Issue #206

## ユーザーストーリー

Docker環境でプロジェクトを管理する運用者として、
セッション作成時にDBに登録されたボリューム名が正しく使用されるようにしたい。
それによって、Docker環境でのworktree作成が正常に動作する。

## 背景

`DockerGitService.getVolumeName(projectId)` が `createWorktree()` から呼ばれる際、`projectName` 引数なしで呼ばれるため、常に `claude-repo-<projectId>` というフォールバック名を返す。しかし実際のボリュームは `cw-repo-<name>` 形式で作成されており、DBの `docker_volume_id` フィールドに正しいボリューム名が記録されている。

## 要件

### 機能要件

- REQ-001: `DockerGitService.createWorktree()` がworktreeを作成する時、システムはDBの `docker_volume_id` フィールドに記録されたボリューム名を使用しなければならない
- REQ-002: `docker_volume_id` がnullの場合、システムは `getVolumeName()` のフォールバック値（`claude-repo-<projectId>`）を使用しなければならない
- REQ-003: `getVolumeName()` メソッドは、オプションとして `docker_volume_id` パラメータを受け取れるようにしなければならない

## 受入基準

1. `docker_volume_id` が設定されたプロジェクトでセッション作成時、DBのボリューム名が使用される
2. `docker_volume_id` がnullのプロジェクトでは、フォールバック名 `claude-repo-<projectId>` が使用される
3. 既存の `createWorktree()` 以外の7メソッド（`deleteWorktree()`, `deleteRepository()`, `getDiffDetails()`, `rebaseFromMain()`, `squashMerge()`, `getCommits()`, `reset()`）も同様に修正される
4. 単体テストで正しいボリューム名が使用されることを検証する

## 影響範囲

- `src/services/docker-git-service.ts`
  - `getVolumeName()` メソッドのシグネチャ変更
  - `createWorktree()` での呼び出し修正
  - その他 `getVolumeName()` を使用するメソッド
- セッション作成API（`docker_volume_id` をDockerGitServiceに渡す）
