# US-003: Docker環境プロジェクトのボリュームIDバリデーション

## 関連Issue

- GitHub Issue #208

## ユーザーストーリー

Docker環境でプロジェクトを管理する運用者として、
`clone_location=docker` のプロジェクトで `docker_volume_id` が未設定の場合に分かりやすいエラーメッセージを受け取りたい。
それによって、不整合な状態のプロジェクトでのセッション作成失敗の原因が明確になる。

## 背景

プロジェクトの `clone_location` が `"docker"` に設定されているにもかかわらず、`docker_volume_id` が `null` のケースが存在しうる。この不整合状態でセッションを作成すると、DockerGitServiceが存在しないボリュームを参照して不明瞭なエラーになる。

## 要件

### 機能要件

- REQ-001: セッション作成APIが呼ばれた時、`clone_location` が `"docker"` かつ `docker_volume_id` が null のプロジェクトに対して、システムは「Dockerボリュームが未設定です」というエラーメッセージと共に400エラーを返さなければならない
- REQ-002: エラーメッセージには、問題の原因と対処方法のヒントを含めなければならない

## 受入基準

1. `clone_location=docker` かつ `docker_volume_id=null` のプロジェクトでセッション作成時、400エラーが返る
2. エラーメッセージが具体的で、対処方法のヒントを含む
3. `clone_location=host` の場合は `docker_volume_id=null` でもエラーにならない
4. `clone_location=docker` かつ `docker_volume_id` が設定済みの場合は正常に処理される
5. 単体テストで各パターンをカバーする

## 影響範囲

- `src/app/api/projects/[project_id]/sessions/route.ts`
  - セッション作成APIにバリデーション追加
