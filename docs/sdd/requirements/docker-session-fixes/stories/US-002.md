# US-002: ネットワークフィルタルール0件時のiptablesチェックスキップ

## 関連Issue

- GitHub Issue #207

## ユーザーストーリー

Docker環境でセッションを起動する運用者として、
ネットワークフィルタリングが有効でもルールが0件の場合はiptablesチェックなしでセッションが起動できるようにしたい。
それによって、iptablesが利用できない環境でもセッション作成が失敗しない。

## 背景

`NetworkFilterService.applyFilter()` は `enabled: true` の場合、フィルタルール数に関係なくiptablesの可用性チェックを実行する。iptablesが利用できないDocker環境では、ルールが0件であっても「iptablesが利用不可のためフィルタリングを適用できません」エラーでPTY作成が失敗する。

適用するルールがない場合、iptablesのチェック自体が不要である。

## 要件

### 機能要件

- REQ-001: `applyFilter()` がフィルタリングを適用する時、有効なルールが0件の場合、システムはiptables可用性チェックをスキップして正常終了しなければならない
- REQ-002: 有効なルールが1件以上存在する場合、システムは従来通りiptables可用性チェックを実行しなければならない

## 受入基準

1. `enabled: true` かつルール0件の場合、`applyFilter()` がエラーなしで正常終了する
2. `enabled: true` かつルール1件以上の場合、従来通りiptables可用性チェックが実行される
3. `enabled: false` の場合、従来通りフィルタリング処理がスキップされる
4. ルール0件で正常終了時、適切なログが出力される
5. 単体テストでルール0件のケースをカバーする

## 影響範囲

- `src/services/network-filter-service.ts`
  - `applyFilter()` メソッドの処理順序変更（ルール取得をiptablesチェックの前に移動）
