# 要件定義: ENCRYPTION_KEY自動生成

## 概要

| 項目 | 内容 |
|------|------|
| フィーチャー名 | ENCRYPTION_KEY自動生成・永続化 |
| 作成日 | 2026-02-26 |
| ステータス | Draft |

## 背景

GitHub PATの暗号化に使用する`ENCRYPTION_KEY`環境変数がDocker環境で未設定のため、PAT登録時に`PATEncryptionError`（500 Internal Server Error）が発生する。現状ではユーザーが`openssl rand -base64 32`で手動生成し、`docker-compose.yml`や`.env`に設定する必要があるが、セットアップドキュメントに記載がなく、ユーザーが気づけない。

## スコープ

### 対象

- サーバー起動時のENCRYPTION_KEY自動生成
- `data/`ディレクトリへのキーファイル永続化（Docker volume経由でコンテナ再起動後も保持）
- 環境変数が明示的に設定されている場合はそちらを優先

### 対象外

- 暗号化アルゴリズムの変更
- 既存の暗号化サービスのリファクタリング
- キーローテーション機能

## EARS記法 要件一覧

### 機能要件

| ID | EARS記法 |
|----|---------|
| FR-001 | サーバー起動時、ENCRYPTION_KEY環境変数が未設定の場合、システムはAES-256-GCM用の32バイト暗号化キーを自動生成しなければならない |
| FR-002 | 自動生成されたキーは、`data/`ディレクトリ内のファイルに永続化されなければならない |
| FR-003 | 次回起動時、キーファイルが存在する場合、システムはファイルからキーを読み込んで使用しなければならない |
| FR-004 | ENCRYPTION_KEY環境変数が明示的に設定されている場合、システムは環境変数の値を優先して使用しなければならない |

### 非機能要件

| ID | EARS記法 |
|----|---------|
| NFR-001 | キーファイルのパーミッションは0600（所有者のみ読み書き可能）でなければならない |
| NFR-002 | キーファイルは`.gitignore`に含まれるディレクトリ（`data/`）に保存されなければならない |
