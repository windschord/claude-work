# US-001: アプリケーションDockerイメージのビルドと公開

## ユーザーストーリー

セルフホストユーザーとして、
Claude Workをdocker runコマンドで簡単に起動したい、
なぜならローカル環境にNode.jsをインストールせずにアプリケーションを使いたいからだ。

## 受入基準

1. `docker pull ghcr.io/<owner>/claude-work:latest` でイメージが取得できる
2. `docker run -p 3000:3000 -v ./data:/data ghcr.io/<owner>/claude-work:latest` でアプリが起動する
3. ブラウザで `http://localhost:3000` にアクセスしてClaude Workが使える
4. コンテナ再起動後もデータ（セッション、プロジェクト設定）が保持される
5. AMD64環境（x86_64 Linux）でも ARM64環境（Apple M系Mac, ARM Linux）でも動作する

## 技術的考慮事項

- `better-sqlite3` はネイティブバイナリのため、ビルドステージでコンパイルが必要
- `node-pty` はネイティブバイナリのため、同様にビルドステージでコンパイルが必要
- マルチステージビルドでビルド時依存とランタイム依存を分離する
- Dockerfileはリポジトリルートに配置する
