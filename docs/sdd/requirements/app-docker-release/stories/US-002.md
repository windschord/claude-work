# US-002: GitHub Releaseの自動作成

## ユーザーストーリー

開発者として、
`git tag v1.0.0 && git push --tags` を実行するだけでGitHub Releaseとリリース済みDockerイメージが公開されてほしい、
なぜなら手動でリリースページを作成したりDockerビルドを実行したりする手間を省きたいからだ。

## 受入基準

1. `v*` 形式のタグ（例: `v0.2.0`）をpushすると自動でGitHub Actionsが起動する
2. GitHub Releasesページに新しいリリースが作成される
3. リリースノートには前回タグからのPR・コミット一覧が自動生成される
4. DockerイメージがGHCRに対応するバージョンタグで公開される
5. `latest` タグも最新リリースのイメージを指す

## 技術的考慮事項

- `softprops/action-gh-release` または GitHubネイティブの `actions/create-release` を使用
- `generate_release_notes: true` で自動リリースノートを有効化
- DockerイメージはGitHub Releaseの作成と同じワークフローで実行する
- パーミッション: `contents: write`, `packages: write` が必要
