# 要件定義: 危険パス警告ダイアログのonBlur発火修正

## 背景

PR #129で実装されたボリュームマウントの危険パス警告ダイアログ（REQ-112）に、入力中の中間状態でダイアログが発火するUXバグが存在する。

### 問題の詳細

ユーザーがhostPathフィールドに`/home/tsk/data`と入力する際、1文字目の`/`が入力された段階で`isDangerousPath("/")`がtrueを返し、`DangerousPathWarning`ダイアログが即座に表示されて入力が中断される。

### 根本原因

`VolumeMountList`コンポーネントのuseEffectが`value`配列の変更ごとに発火し、入力途中の中間状態（`/`のみ）でも`onDangerousPath`コールバックを呼び出している。

## 修正要件

### REQ-112-FIX-001: 危険パス警告の発火タイミング変更

ユーザーがhostPathフィールドへの入力を完了してフォーカスを外した時（onBlur）、システムは危険パス判定を実行しなければならない。入力中（onChange）には危険パス警告ダイアログを表示してはならない。

### REQ-112-FIX-002: インライン警告の維持

入力中のリアルタイムバリデーション（`validateMount`関数によるインライン警告テキスト表示）は、従来通りonChangeで即座に表示しなければならない。これはダイアログではないため、入力を中断しない。

### REQ-112-FIX-003: 既存編集データ読み込み時の非発火

環境編集ダイアログを開いて既存の危険パスデータが読み込まれた時、システムは警告ダイアログを表示してはならない。ユーザーが手動でフィールドを編集（blur）した場合のみ発火する。

### REQ-112-FIX-004: キャンセル後の再入力対応

ユーザーが警告ダイアログで「キャンセル」を選択した後、同じパスを再度入力してblurした場合、システムは再度警告ダイアログを表示しなければならない。

## 影響範囲

- `VolumeMountList`コンポーネント: onBlurハンドラ追加、useEffect削除
- `EnvironmentForm`コンポーネント: 変更なし（onDangerousPathの呼び出し元が変わるだけ）
- `docker-config-validator.ts`: 変更なし
- テストファイル: `VolumeMountList.test.tsx`の該当テストケース更新
