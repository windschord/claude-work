# US-002: 既存Volume選択

## ユーザーストーリー

**私は** ClaudeWorkの管理者として
**〜したい** 環境設定のボリュームマウント時に既存のDocker Volumeを選択できること
**なぜなら** 既存のデータを持つVolumeを再利用したり、複数環境間でVolumeを共有したい

## 受入基準（EARS記法）

### 機能要件

- **REQ-002-001**: DockerClientは、Docker EngineからVolume一覧を取得する `listVolumes()` メソッドを提供しなければならない
  - 各Volumeの名前、ドライバー、作成日時、ラベルを返す
  - `inspectVolume(name)` メソッドで個別Volumeの詳細を取得できる

- **REQ-002-002**: `GET /api/docker/volumes` エンドポイントは、利用可能なDocker Volume一覧を返さなければならない
  - レスポンス形式: `{ volumes: Array<{ name, driver, createdAt, labels }> }`
  - Docker未接続時は適切なエラーレスポンスを返す

- **REQ-002-003**: VolumeMountListコンポーネントにおいて、ユーザーがボリュームマウントを追加する時、システムは「新規Volume作成」と「既存Volumeを使用」の選択肢を表示しなければならない
  - 「既存Volumeを使用」選択時、Docker Volume一覧をドロップダウンで表示する
  - 「新規Volume作成」選択時、現在のhostPathフィールドを維持する

- **REQ-002-004**: Volume名を手動入力する場合、システムはDocker Volume名の制約 `[a-zA-Z0-9][a-zA-Z0-9_.-]*` に準拠しているかバリデーションしなければならない

## 画面/UI要件

### VolumeMountList変更

VolumeMountListの各行に、ソース種別の選択肢を追加:

```
[ホストパス | Docker Volume ▼]
├── ホストパス選択時: 既存のhostPathテキスト入力
└── Docker Volume選択時:
    ├── [既存Volume選択 ▼] (ドロップダウン)
    └── [新規作成: Volume名入力]
```

## データ要件

### Volume一覧APIレスポンス

```typescript
interface DockerVolumeInfo {
  name: string;
  driver: string;
  createdAt: string;
  labels: Record<string, string>;
}

// GET /api/docker/volumes
interface ListVolumesResponse {
  volumes: DockerVolumeInfo[];
}
```
