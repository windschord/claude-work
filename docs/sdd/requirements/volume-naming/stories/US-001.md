# US-001: Volume名の自動生成

## ユーザーストーリー

**私は** ClaudeWorkの管理者として
**〜したい** Docker Volumeに読みやすい名前が自動的に付与されること
**なぜなら** `docker volume ls`でVolume一覧を見た時に、どのプロジェクト/環境に対応するか即座に判別できるようにしたい

## 受入基準（EARS記法）

### 機能要件

- **REQ-001-001**: システムは、入力文字列からDockerVolume名に使用可能なスラッグを生成しなければならない
  - 小文字に変換する
  - スペースおよび特殊文字をハイフンに変換する
  - 連続するハイフンを1つに統合する
  - 先頭・末尾のハイフンを除去する
  - Docker Volume名の制約 `[a-zA-Z0-9][a-zA-Z0-9_.-]*` に準拠する

- **REQ-001-002**: プロジェクトのGitリポジトリ用Volumeを作成する時、システムは `cw-repo-{プロジェクト名スラッグ}` 形式の名前を生成しなければならない

- **REQ-001-003**: Claude Code設定用Volumeを作成する時、システムは `cw-config-{環境名スラッグ}` 形式の名前を生成しなければならない

- **REQ-001-004**: もし生成されたVolume名が既存のDocker Volumeと重複するならば、システムはサフィックス (`-2`, `-3`, ...) を追加して一意な名前を生成しなければならない

- **REQ-001-005**: システムは、生成されたVolume名がDocker Volume名の制約 `[a-zA-Z0-9][a-zA-Z0-9_.-]*` に準拠していることを検証しなければならない

## データ要件

### スラッグ生成ルール

| 入力例 | 出力スラッグ |
|--------|-------------|
| `My Project` | `my-project` |
| `claude-work` | `claude-work` |
| `Test  Project!!` | `test-project` |
| `日本語プロジェクト` | (非ASCII文字除去後、空なら短縮UUIDをフォールバック) |
| `---leading-trailing---` | `leading-trailing` |

### Volume名の形式

| 種別 | 形式 | 例 |
|------|------|-----|
| リポジトリ | `cw-repo-{slug}` | `cw-repo-claude-work` |
| 設定 | `cw-config-{slug}` | `cw-config-default-docker` |

## 備考

- スラッグ生成で全文字が除去される場合（日本語のみのプロジェクト名等）は、短縮UUID（8文字）をフォールバックとして使用する
- Volume名の最大長はDocker側で明確な制限はないが、実用的に64文字以下を推奨
