# US-003: 後方互換性の維持

## ユーザーストーリー

**私は** 既存のClaudeWorkユーザーとして
**〜したい** 既存の `claude-repo-{uuid}` 形式のVolumeが引き続き正常に動作すること
**なぜなら** 既存のデータが失われたり、セッションが動作しなくなることを避けたい

## 受入基準（EARS記法）

### 機能要件

- **REQ-003-001**: もしプロジェクトの `docker_volume_id` が `claude-repo-{uuid}` 形式であるならば、システムはそのVolume名をそのまま使用しなければならない

- **REQ-003-002**: もしプロジェクトに `docker_volume_id` が設定されていないならば、システムは新しい命名規則 (`cw-repo-{slug}`) でVolume名を生成しなければならない

## データ要件

### 互換性マトリクス

| 条件 | Volume名 | 動作 |
|------|---------|------|
| 既存プロジェクト（docker_volume_id設定済） | `claude-repo-{uuid}` | そのまま使用 |
| 新規プロジェクト（docker clone） | `cw-repo-{slug}` | 新規作成 |
| docker_volume_id未設定、フォールバック | `claude-repo-{project_id}` | 既存の動作を維持 |

## 備考

- 既存Volumeの自動リネームは行わない
- 既存プロジェクトの`docker_volume_id`を新命名規則に更新する機能は本Issueのスコープ外
