# Issue #191: テンプレート適用後のUI更新

## 概要

ネットワークフィルタリングのテンプレートを適用した直後にルール一覧UIが自動更新されない問題の修正。

## ユーザーストーリー

Docker環境のネットワークフィルタリング設定画面において、ユーザーがテンプレート適用ダイアログでルールを適用したとき、ダイアログを閉じてから再度開くことなく、ルール一覧がその場で更新されることを期待している。

## 要件（EARS記法）

### 機能要件

- [REQ-01] テンプレート適用APIの呼び出しが成功した後、ネットワークルール一覧が自動的に再取得される。
- [REQ-02] テンプレート適用後に追加されたルールがUI上に即座に反映される。
- [REQ-03] テンプレート適用ダイアログを閉じることなくルール一覧の更新が確認できる。

### 非機能要件

- [REQ-04] テンプレート適用後のUI更新はユーザーの操作を必要としない（自動更新）。

## 受入基準

- テンプレート適用APIが成功した後、`useNetworkFilter` の `refresh` が呼ばれてルール一覧が再取得される。
- ダイアログが開いたままの状態でルール一覧（親コンポーネント）に新しいルールが表示される。
- テンプレート適用失敗時はルール一覧は更新されない。
