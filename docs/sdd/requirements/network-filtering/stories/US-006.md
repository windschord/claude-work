# US-006: Docker環境作成時のデフォルトルール自動適用

## ユーザーストーリー

管理者として、新しいDocker環境を作成した際に、主要サービスへのネットワークアクセスルールが自動的に適用されてほしい。これにより、手動でテンプレートを適用する手間を省き、セキュアなデフォルト設定で即座に使用開始できる。

## 受入基準

### AC-001: Docker環境作成時のフィルタリング自動有効化

- 新規Docker環境作成時、NetworkFilterConfigが `enabled: true` で自動作成される
- HOST環境やSSH環境には適用されない

### AC-002: デフォルトテンプレートルールの自動適用

- 新規Docker環境作成時、以下の全テンプレートルールが自動適用される:
  - Anthropic API: `api.anthropic.com:443`
  - npm: `*.npmjs.org:443`, `*.npmjs.com:443`
  - GitHub: `*.github.com:443`, `*.githubusercontent.com:443`
  - PyPI: `pypi.org:443`, `*.pythonhosted.org:443`
  - Docker Hub: `*.docker.io:443`, `*.docker.com:443`

### AC-003: 環境ごとのカスタマイズ

- 自動適用されたルールは、通常のルールと同様に編集・削除・無効化できる
- 追加のルールも自由に設定可能
- フィルタリング自体を無効にすることも可能

### AC-004: ベストエフォート初期化

- フィルタ初期化に失敗しても、環境作成APIは201（成功）を返す
- 失敗時は警告レベル（warn）のログを出力する（エラーレベルではない）
- ルール適用が完了してからフィルタリングを有効化する（中間状態を防止）

### AC-005: 既存環境への影響なし

- 既存のDocker環境のフィルタリング設定は変更されない
- サーバー起動時にデフォルト環境のルールが上書きされない
