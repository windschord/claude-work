# US-003: デフォルトルールテンプレートの提供

## ユーザーストーリー

**私は** ClaudeWork管理者として
**〜したい** Claude Codeの動作に必要な通信先がプリセットされたデフォルトルールテンプレートを利用したい
**なぜなら** 一からルールを調査・設定する手間を省き、すぐに安全な環境を構築するため

## 受入基準（EARS記法）

### 機能要件

- **REQ-003-001**: 管理者がフィルタリングを初めて有効化した時、システムはデフォルトルールテンプレートの適用を提案しなければならない
- **REQ-003-002**: システムは以下の通信先を含むデフォルトテンプレートを提供しなければならない:
  - Anthropic API (`api.anthropic.com:443`)
  - npm registry (`*.npmjs.org:443`, `*.npmjs.com:443`)
  - GitHub (`*.github.com:443`, `*.githubusercontent.com:443`)
  - PyPI (`pypi.org:443`, `*.pythonhosted.org:443`)
  - Docker Hub (`*.docker.io:443`, `*.docker.com:443`)
- **REQ-003-003**: 管理者がデフォルトテンプレートを適用する時、システムはテンプレートのルールを現在のルールに追加（マージ）しなければならない
- **REQ-003-004**: 管理者がテンプレートから個別のルールを選択/除外できるようにしなければならない
- **REQ-003-005**: もしテンプレートのルールが既存ルールと重複する場合、システムは重複を検出しスキップしなければならない

## 画面/UI要件

- テンプレート適用ダイアログ: チェックボックス付きルール一覧
- 各ルールの説明（どのサービスに必要か）を表示
- 「全選択」「全解除」ボタン

## 備考

- テンプレートの内容はアプリケーションのデフォルト設定としてハードコードする（データベース不要）
- 将来的にカスタムテンプレートの保存・共有機能を検討する可能性があるが、本スコープ外
- テンプレートのルールは定期的に見直しが必要（サービスのドメイン変更等）
