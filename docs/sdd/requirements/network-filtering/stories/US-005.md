# US-005: Docker Compose環境でのフィルタリング対応

## ユーザーストーリー

**私は** ClaudeWork管理者として
**〜したい** Docker Compose環境で運用している場合でも、サンドボックスコンテナにネットワークフィルタリングが適用されてほしい
**なぜなら** Docker Compose環境でもセキュリティレベルを維持し、一貫したネットワーク制御を実現するため

## 受入基準（EARS記法）

### 機能要件

- **REQ-005-001**: ClaudeWorkがDocker Compose環境で動作している時、システムはサンドボックスコンテナ用のフィルタリングネットワークを作成し、メインアプリとの内部通信は維持しなければならない
- **REQ-005-002**: Docker Compose環境において、システムはメインアプリケーションコンテナとサンドボックスコンテナ間の通信を妨げてはならない
- **REQ-005-003**: Docker Compose環境において、システムはサンドボックスコンテナからの外部通信のみをフィルタリング対象としなければならない（メインアプリの通信はフィルタリング対象外）
- **REQ-005-004**: Docker Composeで定義された既存ネットワークとフィルタリング用カスタムネットワークが共存する時、システムはネットワーク間の干渉なく動作しなければならない
- **REQ-005-005**: もしDocker Compose環境の検出に失敗した場合、システムはスタンドアロンDockerと同じフィルタリング方式にフォールバックしなければならない

## データ要件

- Docker Compose環境の検出: 環境変数または`/.dockerenv`ファイルの存在確認
- Docker Composeネットワーク名とフィルタリング用ネットワーク名の競合回避

## 備考

- ClaudeWork自体がDockerコンテナ内で動作する場合、サンドボックスコンテナはDocker-in-Docker（DinD）またはDocker Socket経由で起動される
- フィルタリング用ネットワーク名にはプレフィックス（例: `claudework-filter-`）を付与して既存ネットワークとの衝突を回避する
- Docker Composeのネットワーク設定（`docker-compose.yml`）への変更は不要とする（アプリケーション側で動的に制御）
