## 2. 重複内容の検出

### 2.1 requirements/ と design/ の概要セクション重複

以下のプロジェクトで、requirements の概要・スコープ記述と design の概要・アーキテクチャ概要に内容の重複が見られる:

| プロジェクト | 重複の種類 | 影響度 |
|-------------|-----------|--------|
| disable-host-in-docker | 概要テーブル（フィーチャー名・作成日・ステータス）が両方に存在 | 低 |
| docker-port-volume | requirements のスコープ記述と design のアーキテクチャ概要で同じ機能説明 | 低 |
| docker-named-volume | requirements と design で「バインドマウントから名前付きVolume」の説明が重複 | 低 |

評価: requirements と design で役割が明確に分かれており（requirements=何を/design=どう実装するか）、概要レベルの重複は参照の利便性のため許容範囲。大きな問題ではない。

### 2.2 Docker関連プロジェクト間の重複

Docker関連プロジェクトは10以上存在するが、スコープが異なるため内容の直接的な重複は少ない:

| プロジェクト | スコープ | 重複リスク |
|-------------|---------|-----------|
| docker-port-volume | ポートマッピング・ボリュームマウント設定 | - |
| docker-named-volume | Claude Code設定の名前付きVolume永続化 | volume-naming と Volume概念が近い |
| volume-naming | Volumeの自動命名規則と既存Volume選択 | docker-named-volume と Volume概念が近い |
| volume-delete-options | Volume削除オプション | volume-naming の関連機能 |
| docker-default-remote-clone | Docker環境デフォルト化とリモートclone | disable-host-in-docker とDocker主体化が重複概念 |
| disable-host-in-docker | Docker内でのHOST環境無効化 | docker-default-remote-clone とDocker主体化が重複概念 |

注目すべき重複:
- **docker-named-volume / volume-naming / volume-delete-options**: 3プロジェクトがDocker Volumeの異なる側面を扱っているが、概念が近く、1つの「docker-volume」プロジェクトに統合可能
- **docker-default-remote-clone / disable-host-in-docker**: Docker環境をデフォルトにする方向性が共通。前者はリモートclone機能も含むため完全統合は不適だが、「Docker主体化」の共通コンテキスト文書があると良い

### 2.3 トップレベルの孤立ドキュメント

以下の3ディレクトリが `docs/sdd/` 直下に存在し、`design/`・`requirements/`・`tasks/` の標準構造に従っていない:

| ディレクトリ | 内容 | 問題 |
|-------------|------|------|
| `docs/sdd/docker-migration-fix/` | DBマイグレーション修正とDocker起動時自動実行 | design/requirements/tasks 構造外 |
| `docs/sdd/docker-primary-deploy/` | Docker Composeプライマリデプロイ化 | design/requirements/tasks 構造外 |
| `docs/sdd/dockerode-migration/` | Docker CLI -> Dockerode ライブラリ移行 | design/requirements/tasks 構造外 |

これらはいずれのインデックスファイルにも登録されていない。各ディレクトリ内に design.md / requirements.md / tasks.md が1ファイルずつ格納されている旧形式。

## 5. 推奨アクション（優先度順）

### 優先度: 高

1. **インデックスファイルの全面更新** - 3つのインデックスファイルに未記載のプロジェクトが多数ある（design: 18件、requirements: 16件、tasks: 18件未記載）。インデックスが信頼できる一覧として機能していない
2. **ステータス不整合の修正** - 完了しているのに「進行中」「計画中」のままの3件（disable-host-in-docker, network-filtering, add-worktree-option）を修正
3. **トップレベル孤立ドキュメントの移動** - `docker-migration-fix/`、`docker-primary-deploy/`、`dockerode-migration/` を標準構造に移動するか archive/ に移す

### 優先度: 中

4. **肥大化ファイルの分割** - 特に `design/settings-ui/index.md`(879行)、`tasks/docker-port-volume/tasks.md`(716行)、`design/disable-host-in-docker/design.md`(691行) をコンポーネント別・フェーズ別に分割
5. **Docker Volume関連プロジェクトの統合検討** - docker-named-volume / volume-naming / volume-delete-options を1つの「docker-volume」プロジェクトに統合

### 優先度: 低

6. **完了済み機能で design のみ存在し tasks がないプロジェクトの整理** - 初期のプロジェクト（claude-interaction, core, git-operations等）はSDD導入前の実装のため tasks が存在しない。これらは archive/ への移動候補
7. **requirements と design の概要重複の軽減** - 概要セクションを「要件定義書を参照」のリンクに置き換える。ただし影響は小さい
