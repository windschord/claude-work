# US-006: Docker環境作成時のデフォルトネットワークフィルタリング自動適用

## ユーザーストーリー

管理者として、Docker環境を新規作成した際に、デフォルトのネットワークフィルタリングルールが自動的に適用され、フィルタリングが有効化されることで、手動設定なしにセキュアな環境を利用開始したい。

## 背景

- 新規Docker環境作成時にフィルタリングが未設定だと、ユーザーが手動でルール設定するまで無防備な状態になる
- デフォルトテンプレート（Anthropic API, npm, GitHub, PyPI, Docker Hub）を自動適用することで、即座にセキュアな状態で利用可能にする
- Proxy同期はコンテナ起動時にDockerAdapterが自動で行うため、環境作成時点では不要

## 受入基準

| # | 基準 | EARS記法 |
|---|------|----------|
| AC-1 | Docker環境作成時に全デフォルトテンプレートがルールとして自動適用される | イベント: Docker環境(type=DOCKER)が作成された時、システムは全デフォルトテンプレートのルールを自動適用しなければならない |
| AC-2 | フィルタリングがデフォルトで有効化される | イベント: デフォルトテンプレートのルール適用が成功した時、システムはフィルタリングを有効化しなければならない |
| AC-3 | HOST/SSH環境では自動適用しない | 条件: 環境タイプがHOSTまたはSSHの場合、システムはネットワークフィルタリングの自動適用を行ってはならない |
| AC-4 | フィルタ初期化失敗時も環境作成は成功する | イベント: デフォルトルール適用またはフィルタリング有効化が失敗した時、システムは警告ログを出力し、環境作成自体は成功(201)を返さなければならない |
| AC-5 | テンプレート適用失敗時はフィルタリング有効化をスキップする | イベント: テンプレート適用(applyTemplates)が失敗した時、システムはフィルタリング有効化(updateFilterConfig)を実行してはならない（ルール0件で有効化する中間状態を防止） |
