# US-004: フィルタリング状態の確認・モニタリング

## ユーザーストーリー

**私は** ClaudeWork管理者として
**〜したい** 現在適用されているフィルタリングルールと通信の状態を確認したい
**なぜなら** ルールが正しく適用されているか検証し、必要な通信がブロックされていないか確認するため

## 受入基準（EARS記法）

### 機能要件

- **REQ-004-001**: 管理者がセッション詳細画面を開いた時、もしフィルタリングが有効ならば、システムはフィルタリングの適用状態（有効/無効、ルール数）を表示しなければならない
- **REQ-004-002**: 管理者が環境設定画面でフィルタリング状態を確認した時、システムは現在稼働中のコンテナに適用されているルール数を表示しなければならない
- **REQ-004-003**: もしフィルタリングの適用に失敗したコンテナがある場合、システムはエラー状態を環境設定画面に表示しなければならない
- **REQ-004-004**: 管理者がフィルタリングのテスト機能を実行した時、システムは指定したドメイン/IPへの通信が許可されるかブロックされるかを判定し結果を表示しなければならない

## 画面/UI要件

- 環境設定画面: フィルタリングステータスバッジ（有効: 緑、無効: グレー、エラー: 赤）
- セッション一覧: フィルタリング適用状態のアイコン表示
- 通信テスト機能: ドメイン/IP入力 + ポート入力 + テスト実行ボタン + 結果表示

## 備考

- 通信テストはdry-run的にルールをチェックするもので、実際の通信は発生させない
- 詳細なトラフィックログ（ブロックされた通信のログ等）は将来の拡張として検討
