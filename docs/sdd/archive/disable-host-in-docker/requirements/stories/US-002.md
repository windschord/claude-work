# US-002: ALLOW_HOST_ENVIRONMENTによるHOST環境有効化

## ユーザーストーリー

**私は** ClaudeWorkの上級ユーザーとして
**~したい** Dockerコンテナ内でもHOST環境を使用できるようにオプトインできる
**なぜなら** 特殊なデプロイ構成（Docker-in-Dockerやホストネットワークモードなど）でHOST環境が必要な場合があるため

## 受入基準（EARS記法）

### AC-001: 環境変数による有効化

- 環境変数 `ALLOW_HOST_ENVIRONMENT=true` が設定されている時、システムはDockerコンテナ内でもHOST環境の作成・利用を許可しなければならない
- `ALLOW_HOST_ENVIRONMENT` が未設定または `true` 以外の値の場合、HOST環境は無効化されたままでなければならない

### AC-002: 有効化時の動作

- `ALLOW_HOST_ENVIRONMENT=true` が設定されている時、環境作成APIはtype=HOSTのリクエストを受け付けなければならない
- `ALLOW_HOST_ENVIRONMENT=true` が設定されている時、HOST環境を指定したセッション作成を許可しなければならない
- `ALLOW_HOST_ENVIRONMENT=true` が設定されている時、UIでHOSTタイプが選択可能でなければならない

### AC-003: docker-compose.ymlでの設定

- `docker-compose.yml` に `ALLOW_HOST_ENVIRONMENT` 環境変数のコメントアウトされた設定例が記載されなければならない
