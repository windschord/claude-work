# US-003: Docker内でのデフォルト環境切り替え

## ユーザーストーリー

**私は** ClaudeWorkの管理者として
**~したい** Dockerコンテナ内ではデフォルト環境が自動的にDOCKER環境になる
**なぜなら** HOST環境が無効な状態でデフォルト環境がHOSTのままだと、セッション作成が失敗するため

## 受入基準（EARS記法）

### AC-001: デフォルト環境の切り替え

- Dockerコンテナ内で動作している時（かつALLOW_HOST_ENVIRONMENT=trueでない場合）、サーバー起動時にデフォルト環境をDOCKER環境に設定しなければならない
- 既存のHOSTデフォルト環境がある場合、そのis_defaultフラグをfalseに変更しなければならない

### AC-002: DOCKER環境の自動作成

- Dockerコンテナ内で動作している時、デフォルトのDOCKER環境（id='docker-default'）が存在しない場合は自動作成しなければならない
- 自動作成されるDOCKER環境はデフォルト環境（is_default=true）として設定されなければならない

### AC-003: 非Docker環境での動作保持

- ホスト環境で直接動作している場合、デフォルト環境の設定は現行動作（HOST環境がデフォルト）を維持しなければならない
