# データ整合性要件

> 非機能要件カテゴリ: データ整合性

## 概要

プロジェクトと環境の紐付けにおいて、データの整合性を保証し、不正な状態の発生を防止する。

## 要件一覧

### NFR-DI-001: APIレベルでの制約強制

**EARS記法**: 環境の削除制限および環境変更制限は、APIレベルで強制されなければならない（UIバイパス対策）

**測定基準**:
- 指標: API直接呼び出しでの制約有効性
- 目標値: UI経由/直接呼び出しの両方で制約が機能
- 測定方法: APIテストでの検証

**関連する機能要件**: FR-005, FR-006, FR-012

**優先度**: 高

### NFR-DI-002: 環境変更の原子性

**EARS記法**: プロジェクトの環境変更処理は、セッション存在チェックと環境ID更新をトランザクション内で実行しなければならない

**測定基準**:
- 指標: レースコンディションの発生有無
- 目標値: 並行リクエストでも不整合が発生しない
- 測定方法: 並行テストでの検証

**関連する機能要件**: FR-012

**優先度**: 高

### NFR-DI-003: 環境参照の一貫性

**EARS記法**: セッション作成からWebSocket接続まで、システムは一貫してプロジェクトのenvironment_idを参照しなければならない

**測定基準**:
- 指標: 環境解決パスの統一性
- 目標値: 全経路でproject.environment_idを使用
- 測定方法: コードレビューおよびテスト

**関連する機能要件**: FR-014, FR-015

**優先度**: 高
