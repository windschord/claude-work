# 要件定義: プロジェクト-環境バインディングの一元化

## 概要

| 項目 | 内容 |
|------|------|
| フィーチャー名 | project-environment-binding |
| 作成日 | 2026-02-27 |
| ステータス | ACTIVE |

## 背景

現在、ClaudeWorkではセッション作成時に実行環境（Docker環境/コンテナ）を選択できる設計になっており、プロジェクトとセッションの両方に`environment_id`が存在する。環境選択の4段階フォールバックロジックが複雑で、どの環境で実行されるかが暗黙的に決定されるケースがある。

この変更により、環境選択をプロジェクト単位に一元化し、以下を実現する:
- プロジェクト作成時に環境を必須で指定
- セッションは常にプロジェクトの環境を継承
- 環境の削除・変更に適切な制約を設ける

## スコープ

### 対象

- プロジェクト作成画面での環境選択UI（必須化）
- プロジェクト設定画面での環境表示・条件付き変更
- セッション作成モーダルからの環境選択UI削除
- セッション作成APIの環境決定ロジック簡素化
- 環境削除APIの使用中チェック追加
- WebSocket接続時の環境解決ロジック簡素化

### 対象外

- 環境（Docker/Host/SSH）自体の機能変更
- 新しい環境タイプの追加
- 既存データのマイグレーション（新規設計のみ）
- Docker環境の設定項目（ポート、ボリューム等）の変更

## EARS記法 要件一覧

### 機能要件

| ID | EARS記法 |
|----|---------|
| FR-001 | プロジェクト作成時、システムは実行環境（environment_id）の指定を必須としなければならない |
| FR-002 | プロジェクト作成APIにenvironment_idが未指定の場合、システムはバリデーションエラー（400）を返さなければならない |
| FR-003 | プロジェクト作成画面において、システムは利用可能な環境の選択ドロップダウンを表示しなければならない |
| FR-004 | 環境が1つも登録されていない場合、プロジェクト作成画面はプロジェクト作成を無効化し、理由を表示しなければならない |
| FR-005 | 環境削除APIが呼び出された時、システムは対象環境を使用しているプロジェクトが存在するかチェックしなければならない |
| FR-006 | もし対象環境を使用しているプロジェクトが1つ以上存在するならば、システムは削除を拒否しエラー（409 Conflict）を返さなければならない |
| FR-007 | 環境削除エラーレスポンスには、使用中のプロジェクト名の一覧を含めなければならない |
| FR-008 | 環境管理UIにおいて、プロジェクトで使用中の環境には削除ボタンを無効化し、使用中であることを表示しなければならない |
| FR-009 | プロジェクト設定画面において、システムは紐付いた環境名を表示しなければならない |
| FR-010 | プロジェクト内のセッションが0件の場合、プロジェクト設定画面の環境変更ボタンを有効化しなければならない |
| FR-011 | もしプロジェクト内にセッションが1件以上存在するならば、システムは環境変更ボタンを無効化し、「セッションが存在するため変更できません」と表示しなければならない |
| FR-012 | プロジェクトの環境変更APIが呼び出された時、もしプロジェクト内にセッションが存在するならば、システムは変更を拒否しエラー（409 Conflict）を返さなければならない |
| FR-013 | セッション作成モーダルにおいて、システムは環境選択ドロップダウンを表示してはならない |
| FR-014 | セッション作成APIが呼び出された時、システムはリクエストのenvironment_idパラメータを無視し、プロジェクトのenvironment_idを使用しなければならない |
| FR-015 | WebSocket接続時、システムはセッションのプロジェクトのenvironment_idを使用して環境を解決しなければならない |

### 非機能要件サマリー

| ID | EARS記法 |
|----|---------|
| NFR-001 | 環境の削除制限・変更制限はAPIレベルで強制されなければならない（UIバイパス対策） |
| NFR-002 | プロジェクト作成画面の環境選択は、現在利用可能な環境のみを表示しなければならない |
| NFR-003 | 環境変更の可否判定は、セッション数のカウントクエリで行い、100ms以内に応答しなければならない |
