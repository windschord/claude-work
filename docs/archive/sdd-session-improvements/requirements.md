# 要件定義: セッション管理機能の改善

## 概要

ClaudeWorkのセッション管理機能を改善し、ユーザビリティを向上させる。
主な改善点は以下の3つ：
1. セッション削除機能のUI実装
2. セッション名の自動生成
3. セッション作成UIの簡略化

## ユーザーストーリー

### ストーリー1: セッションの削除

**私は** ClaudeWorkのユーザーとして
**〜したい** 不要になったセッションを削除
**なぜなら** プロジェクト内のセッション一覧を整理し、必要なセッションを見つけやすくしたいから

#### 受入基準（EARS記法）

- REQ-001: ユーザーがセッションカードの削除ボタンをクリックした時、システムは削除確認ダイアログを表示しなければならない
- REQ-002: 削除確認ダイアログにおいて、システムは削除対象のセッション名を表示しなければならない
- REQ-003: ユーザーが削除確認ダイアログで「削除」を選択した時、システムはセッションを削除しなければならない
- REQ-004: セッション削除時、システムは関連するworktreeも削除しなければならない
- REQ-005: ユーザーが削除確認ダイアログで「キャンセル」を選択した時、システムはセッションを削除せずダイアログを閉じなければならない

### ストーリー2: セッション名の自動生成

**私は** ClaudeWorkのユーザーとして
**〜したい** セッション名を自動生成してほしい
**なぜなら** セッション名を毎回考えるのは手間であり、一意で覚えやすい名前が欲しいから

#### 受入基準（EARS記法）

- REQ-006: セッション作成時にセッション名が未入力の場合、システムは「形容詞-動物名」形式の名前を自動生成しなければならない
- REQ-007: 自動生成される名前は、英小文字とハイフンのみで構成されなければならない
- REQ-008: 自動生成される名前は、同一プロジェクト内で一意でなければならない

### ストーリー3: セッション作成UIの簡略化

**私は** ClaudeWorkのユーザーとして
**〜したい** シンプルなセッション作成フォームを使いたい
**なぜなら** 複数セッション作成は不要であり、シンプルな操作で素早くセッションを開始したいから

#### 受入基準（EARS記法）

- REQ-009: セッション作成フォームにおいて、セッション名入力は任意でなければならない
- REQ-010: セッション作成フォームにおいて、システムはセッション数選択UIを表示してはならない
- REQ-011: セッション作成フォームにおいて、システムは単一セッション作成のみをサポートしなければならない

## 非機能要件

- NFR-001: セッション削除は2秒以内に完了しなければならない
- NFR-002: セッション名自動生成は100ms以内に完了しなければならない

## 関連する既存API

| API | メソッド | 状態 |
|-----|---------|------|
| /api/projects/[project_id]/sessions | POST | 変更必要（名前を任意に） |
| /api/projects/[project_id]/sessions/bulk | POST | 削除 |
| /api/sessions/[id] | DELETE | 変更なし（既に実装済み） |
